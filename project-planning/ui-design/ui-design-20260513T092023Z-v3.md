> **Reading note.** Variant **3 of 3** for PRD-001 (multilingual CRUD) on Redirect Manager. The named direction is **"Workspace Frame"** — the Full Page surface becomes a workspace, not a list-with-picker. A sticky left **language sidebar** stacks every site language as a navigation entry. Map detail switches versions via **tabs at the top**. Create-version uses a **right-edge drawer**. Delete uses a **consequence-preview** panel that names what data goes and what stays. This is the moderate-divergence outer edge: a new layout grammar (sidebar + tabs + drawer) composed entirely from Blok primitives. All component names below reference real Blok primitives or composite blocks via `@blok/*` registry slugs. All colors and spacing use Blok semantic tokens — never raw hex, never invented token names. PRD-000's Operator Console aesthetic survives (dense rows, monospace for paths, badge-based status grammar); the workspace pattern adds verbs on top, not a repaint.

## 1. Design context & assumptions

### 1.1 — Audience

PRD-001 inherits PRD-000's three personas; multilingual changes what each can do, not who they are.

- **Marketers** — author redirects in their working language (often non-`en`). The workspace pattern matters because they switch languages many times per session — anything that hides the active language behind a dropdown costs them muscle memory. The sidebar makes the active language a **visible badge of state**, not a hidden picker selection.
- **Site administrators** — manage multilingual sites. They care about per-language coverage at a glance. The matrix-cell map list pattern (§ 3.2) is built for this audience: scan a column down for "which maps have `de`."
- **Migration operators** — re-import multilingual JSON bundles. The grouped-by-language import preview (§ 3.5) matches the mental model of the export format (each language is its own collapsible group).

### 1.2 — Goals (mapped to PRD US-1 through US-13)

| Workspace Frame promise | Maps to |
|---|---|
| Language is **always visible** — never hidden behind a dropdown | US-1, US-12 (Dashboard scope toggle) |
| Switching languages takes **one click** from anywhere in the workspace | US-1, US-2 |
| The active language is **part of the chrome**, not a control state | US-3 (toast names the language; chrome reflects it too) |
| Cross-language coverage is **scannable** like a heatmap | US-2 |
| Creating a version preserves the **list context** (no modal eclipse) | US-6, US-7 |
| Deleting names the **consequence** before commit | US-8 (strict no-default per ADR-0018) |
| The Context Panel banner is **Pages-language-aware** | US-13 (PRD-002 supersedes matcher) |

### 1.3 — Constraints

- **Blok mandatory.** Marketplace public-app submission requires `theme.json` + canonical `@blok/*` components (`blok-marketplace-integration` skill, submission checklist).
- **Mode A iframe.** No top-level navigation; SDK-only cross-frame.
- **Iframe viewport budgets** (per `blok-marketplace-integration` skill):
  - Full Page: 1024–1920 px wide × 600–1080 px tall. Sidebar fits — see § 5.3 for narrow-Full-Page collapse.
  - Pages Context Panel: 320–400 px × scrollable. **No sidebar, no tabs, no drawer.** Banner-only changes per architecture § 2.2.
  - Dashboard Block: 300–800 px × 200–400 px. Inline scope toggle replaces a sidebar.
- **WCAG 2.1 AA** (NFR-A1) — every new pattern has a documented keyboard story and ARIA role.
- **PRD-000 visual continuity** — same Blok foundation, same theme tokens, same Nova preset. PRD-000 winning POC (`pocs/poc-v1/`) already runs a left rail with collection/site pickers — Workspace Frame extends that rail with a language stack rather than introducing a foreign element.

### 1.4 — Reference & inspiration

Named patterns this direction draws on (not copied — composed from Blok primitives):

- **Sitecore Content Editor's content tree** — left-rail navigation stack as the operator's "where am I?" anchor. The language stack borrows the same vertical-list-with-active-row pattern.
- **VS Code's primary sidebar + activity bar** — sidebar entries persist; the editor area swaps. Maps cleanly onto "language stays selected; map detail swaps when you pick a different map."
- **Sitecore Authoring Tool's language tabs in item editor** — operator switches versions inline at the top of the detail pane. PRD-001's tabs (§ 3.3) reuse this muscle memory directly.
- **Figma / Linear's right-edge drawer for entity creation** — the drawer slides over context, but the operator can see the list behind it through a scrim. Preserves orientation when creating a new language version.
- **GitHub's "Files changed" delete confirmation pattern** — show what changes before the operator clicks Confirm. Underpins § 3.4 consequence-preview.

### 1.5 — Answers to clarifying questions (carried from PRD-001 Discovery)

Documented in PRD § 11 UX Considerations and § 14 Open Questions. V3-relevant locked decisions:

- Picker state persists via `localStorage`, shared across Full Page + Dashboard (ADR-0017). **V3 surfaces this as a sidebar nav, not a dropdown.**
- Create-version has two paths (empty + copy-from), no name input (ADR-0016). **V3 surfaces both in one drawer.**
- Delete confirmation has strict no-default radios (ADR-0018). **V3 evolves this into a consequence-preview panel where the radios live alongside the count of data they will remove.**
- `__Display name` editing has a discoverable affordance with Enter-saves / blur-does-NOT-commit semantics (AC-9.4). **V3 uses click-to-edit on the title — affordance is the title itself.**
- Context Panel banner reads `pageInfo.language` (AC-13.2). **V3 integrates it into a workspace-style topbar of the Context Panel.**

### 1.6 — Assumptions made (V3-specific, not in PRD)

- **Sidebar width budget — 220 px** at the default Full Page viewport (1280 px wide). Tight enough to keep ≥1000 px for the workspace area; wide enough to fit `nativeName` + `displayName` for languages like `Schweizerdeutsch (de-CH)` without truncation. Verify at clickdummy; adjust to 240 px if any common language overflows.
- **Tab visual density** — text-only tabs (no icons) keeps Tab-row short enough to fit 6+ languages in 800 px of detail-pane width without horizontal scroll. Above 6, tabs scroll horizontally (§ 5.2).
- **Drawer width — 40%** of viewport at desktop (≈520 px at 1280 px), full-width on Context-Panel-narrow viewports (drawer is Full Page only; Context Panel does not host it).
- **Consequence-preview panel** is a section inside the standard `@blok/alert-dialog`, not a new top-level pattern. It's how the dialog body is composed — not a new component.
- **Sidebar collapses to a top-bar `@blok/select` below 900 px viewport** — Marketplace Full Page rarely goes that narrow, but the iframe is host-driven so the collapse is required (NFR-C1).

## 2. Design direction

### 2.1 — Visual direction: "Workspace Frame"

The Full Page surface is a **workspace**. Three regions, always visible:

```
┌──────────────────────────────────────────────────────────────────────────┐
│  Topbar — site/collection breadcrumb, global actions (Import / Export / New map)
├──────────┬───────────────────────────────────────────────────────────────┤
│ Language │  Workspace area                                               │
│ Sidebar  │  ┌─────────────────────┬─────────────────────────────────────┐ │
│ ──────── │  │ Map list (matrix)   │ Map detail (tabs at top)            │ │
│ • English│  │ ┌─ name        cov ┐│  ┌ en | de* | fr | (+ create) ──┐   │ │
│ • Deutsch│  │ │ Marketing  ●●○●  ││  │                              │   │ │
│   FR     │  │ │ Legacy URLs●○●●  ││  │ [mapping table]              │   │ │
│   IT     │  │ │ ...              ││  │                              │   │ │
│   JA     │  │ └──────────────────┘│  └──────────────────────────────┘   │ │
│          │  └─────────────────────┴─────────────────────────────────────┘ │
└──────────┴───────────────────────────────────────────────────────────────┘
```

The **active language** in the sidebar is the operator's anchor — visually emphasized (left accent bar `bg-primary`, `text-foreground` row, `bg-accent/40` row background). All other sidebar entries are dim (`text-muted-foreground`, no row background). Clicking another language is one click; the sidebar selection updates, the matrix recomputes which column is highlighted, the detail-pane tabs re-order so the active language is leftmost.

**Type, color, spacing — inherited from PRD-000 / Blok semantic tokens. No new tokens.**

- **Typography:** Geist Sans (already in Nova preset). Sidebar rows use `text-sm` for `nativeName` + `text-xs text-muted-foreground` for ISO code. Tab labels use `text-sm font-medium`. Workspace title (the map name as click-to-edit) uses `text-2xl font-semibold`. Mapping table cells stay monospace (`font-mono text-sm`) — PRD-000 continuity.
- **Color:** all Blok semantic tokens — `bg-background` (workspace), `bg-card` (sidebar + detail pane), `bg-muted` (sidebar row hover), `bg-accent/40` (active row tint), `text-foreground`, `text-muted-foreground`, `border-border`, `bg-primary` (active accent bar, primary button), `bg-destructive/10` + `text-destructive` (consequence-preview "removes" section), `bg-success/10` + `text-success` (consequence-preview "preserves" section). Dark + light pair both verified.
- **Spacing:** `gap-3` between sidebar rows, `gap-4` between workspace columns, `p-4` workspace padding, `p-3` sidebar-row padding. Same scale as PRD-000.
- **Radius / shadow:** `rounded-md` (matches PRD-000 button radius). Sidebar floats with `shadow-sm`; drawer uses `shadow-lg` (the only place a heavy shadow is acceptable — the drawer is meant to feel detached from the workspace plane).
- **Motion:** drawer slides in with `transition-transform duration-200 ease-out`; tab swap is instant content swap (no fade — keeps the Sitecore Authoring Tool muscle memory honest); consequence-preview reveals via `transition-opacity duration-150`. `prefers-reduced-motion: reduce` disables all three.

### 2.2 — Layout & structure (per extension point)

**Full Page (`/full-page`).** Three-region workspace per § 2.1. Order in DOM: topbar (`<header>`) → main flex row with sidebar (`<nav aria-label="Languages">`) and workspace area (`<main>` containing list `<section>` + detail `<section>`).

**Dashboard Block (`/dashboard-widget`).** **No sidebar** — too narrow. Inline scope toggle on each tile (`@blok/toggle-group` or composed `<button role="radio">` per blok-components skill) — operator picks between "Current language only" and "All languages". Persisted choice in `localStorage` under the same key as the picker (ADR-0017 extension, V3-only — falls back gracefully if other surfaces don't read this key). Twin tiles ("current" + "all") are explicitly rejected as a V3 choice: the toggle is the more compact pattern for a 300–800 px widget.

**Context Panel (`/context-panel`).** **No workspace pattern.** Panel is too narrow (320–400 px) for sidebar/tabs/drawer. The Context Panel changes in PRD-001 are: (a) the static banner becomes a **workspace-styled topbar banner** — same Pages-aware copy as the architecture pins, but rendered as a `@blok/alert` with `variant="info"` and a left accent bar matching the sidebar pattern in Full Page. Visual consistency without re-importing the workspace structure. Matcher unchanged (R-5 accepted).

### 2.3 — Key components & patterns (Blok slugs)

| Workspace region | Composed from |
|---|---|
| Topbar | `@blok/topbar` (PRD-000 carryover) + `@blok/breadcrumb` + `@blok/button` (Import / Export / New map actions) |
| **Language sidebar** | `@blok/navigation-stack` (the canonical Blok vertical-list-nav primitive) — each row is a button with `aria-current="page"` on the active language; left accent bar styled via `border-l-2 border-primary` |
| Map list (matrix) | `@blok/scroll-area` wrapping a table built from `@blok/table` primitives; each row's coverage matrix is a horizontal flex of `@blok/badge` cells, one per language |
| **Detail tabs** | `@blok/tabs` (real Blok primitive) — tabs reorder so active language is leftmost; languages without versions render a special tab with `data-state="missing"` and "(create)" affordance |
| Click-to-edit title | `@blok/editable` (Blok primitive — see blok-components skill, "Forms & input") |
| **Create-version drawer** | `@blok/sheet` (Blok primitive — `sheet` is Blok's drawer name; see blok-components catalog under "Overlays & dialogs"). Side="right". Two CTAs at top, source-language picker inline below when Copy from chosen. |
| **Consequence-preview delete** | `@blok/alert-dialog` (real Blok primitive) composed with two `@blok/alert` panels inside: one `variant="destructive"` for "removes", one `variant="success"` for "preserves". Radios use `@blok/radio-group`. |
| Display-name click-to-edit | `@blok/editable` (Blok primitive). Enter saves, Escape cancels, blur does NOT commit (AC-9.4). |
| Import preview groups | `@blok/accordion` (real Blok primitive) — one group per language; collapsible. Inside each group, `@blok/table` rendering per-map diffs. |
| Context Panel banner | `@blok/alert` (real Blok primitive) with `variant="info"` |
| Toasts | `@blok/sonner` (PRD-000 carryover; named language in toast copy per AC-3.4) |
| Skeletons | `@blok/skeleton` (PRD-000 carryover) for sidebar + matrix loading |
| Empty / error states | `@blok/empty-states`, `@blok/error-states` (PRD-000 carryover) |

**Components NOT in the default Marketplace quickstart** (per blok-components skill — must be `shadcn add`-ed before /implement): `navigation-stack`, `tabs`, `sheet`, `editable`, `accordion`, `radio-group`, `alert-dialog`. The Lead Developer adds these in Tranche 1 setup.

## 3. Key screens and flows

### 3.1 — Full Page: home with sidebar + matrix list

```
┌──────────────────────────────────────────────────────────────────────────────┐
│ SneakerCo / MainSite                                  [Import][Export][+ New map]│  ← topbar
├──────────┬───────────────────────────────────────────────────────────────────┤
│ Languages│ Redirect maps · MainSite                            🔍 [search]   │
│          │ ┌──────────────────────┬─────────────────────────────────────────┐│
││English  │ │ Name              en de fr ja  · updated · type · count        ││  ← matrix list
││Deutsch  │ │ Marketing camps   ● ● ○ ●     · 3d ago  · 301  · 12            ││
│ Français │ │ Legacy URLs       ● ○ ● ●     · 2w ago  · 302  · 28            ││
│ Italiano │ │ A/B variants      ● ● ● ○     · 1d ago  · 301  · 4             ││
│ 日本語   │ │ …                                                              ││
│          │ └──────────────────────┴─────────────────────────────────────────┘│
│ + Add    │                                                                   │
│  language│                                                                   │
└──────────┴───────────────────────────────────────────────────────────────────┘
```

Coverage matrix glyphs:
- `●` (`bg-success`, monochrome dot) — version exists in this language **with non-empty mappings**
- `◐` (half-filled, `bg-warning`) — version exists but `UrlMapping` is empty (US-5 "empty version" state)
- `○` (outline only, `border-muted-foreground`) — no version exists (US-4 "no version" state)

Clicking a matrix cell jumps to the map's detail pane with that language's tab pre-selected (or the "missing" tab if `○`). Hovering a matrix row dims the other rows so the operator's eye stays on one row.

**Active-language column highlight.** When the operator's sidebar selection is `de`, the `de` column in every matrix row gets a `bg-accent/20` tint behind the dot — operator's eye lands on the active language across all maps simultaneously. This is the matrix's payoff: status of every map in the current language is one glance away.

**Long-list relationship-visibility check (per signature behavior).** Matrix list keeps "map name on the left" and "coverage cells on the right" on the same row. At 100+ maps with `@blok/scroll-area` virtualization (react-virtuoso, ADR-0012 carryover), the row binding is preserved by the row container — no parallel-scrolling list. The matrix scales because each cell is 32 px wide; 6 languages = 192 px of matrix per row, easily fits beside name + meta in 600 px of list width. Beyond 6 languages: matrix scrolls horizontally inside the row (rare — most sites have ≤6 languages).

**Sidebar "+ Add language" affordance.** Disabled by default with a tooltip *"Add a new site language via Sitecore Authoring."* — the Redirect Manager app does not author new languages on the tenant; this is a navigation hint, not a write. When the SDK exposes `xmc.sites.addSiteLanguage` (future PRD, not PRD-001), the affordance becomes live.

### 3.2 — Map detail: tabs at top, click-to-edit title

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ ◀ Marketing campaigns                       [✕ Delete] [Save] [Discard]     │  ← title is clickable to edit
│ ┌────────────────────────────────────────────────────────────────────────┐  │
│ │  en │ de* │ fr (create) │ ja                                            │  │  ← tabs
│ └────────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│ ┌─ versioned ───────────────────────────────────────────────────────────┐   │
│ │ Mappings                                                              │   │
│ │ ┌──────────────────────────┬──────────────────────────┬─────────────┐ │   │
│ │ │ Source                   │ Target                   │       │     │ │   │
│ │ │ /old/winter-sale         │ /promotions/winter       │ ✎  ✕  │     │ │   │
│ │ │ /old/spring-sale         │ /promotions/spring       │ ✎  ✕  │     │ │   │
│ │ └──────────────────────────┴──────────────────────────┴─────────────┘ │   │
│ │ [+ Add mapping]                                                        │   │
│ └────────────────────────────────────────────────────────────────────────┘   │
│ ┌─ shared across all languages ─────────────────────────────────────────┐   │
│ │ Redirect type: [301 ▾]  ☑ Include virtual folder  ☐ Preserve qs        │   │
│ │ ⓘ These settings apply to every language version of this map.          │   │
│ └────────────────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────────────┘
```

**Tab states:**
- `en` — version exists, currently inactive (operator viewing `de*`)
- `de*` — version exists, currently active (asterisk = dirty edit indicator per AC-3.5)
- `fr (create)` — no version exists; clicking opens the create-version drawer (§ 3.3) with `fr` pre-selected as target
- `ja` — version exists, inactive

**Why tabs + sidebar both.** This is the apparent redundancy V3 explicitly chooses: the sidebar tracks the *operator's working language* (sticky across maps); the tabs track the *map's languages* (per-map state). They are different anchors for different decisions. The sidebar selection drives which tab opens first when you pick a new map; clicking a tab does NOT change the sidebar (it's a per-map navigation, not a workspace mode shift). This separation is what the workspace metaphor buys — your "where am I" stays stable while you explore a single map's versions.

**Click-to-edit title** (per § 1.5 click-to-edit promise). Hovering the map name reveals a thin `border-b border-dashed border-border` underline. Clicking turns the title into a `@blok/editable` input bound to current-language `__Display name`. Enter or `Save` button → commits via `updateItem`. Escape or `Cancel` → discards. Blur does NOT commit (AC-9.4). When `__Display name` is empty, the title renders italic gray placeholder `_(no display name for [language] — add one)_` per AC-9.2.

**Shared-vs-versioned visual grouping.** The detail pane is split into two `@blok/card` sections: "Mappings" (versioned content) on top, and "Settings — shared across all languages" (the flag + redirect-type cluster) below. The lower section carries a persistent `@blok/alert` hint clarifying the shared scope. This kills the AC-3.2 inline-toast pattern in favor of always-visible grouping — operator never has to discover the scope mid-edit because the layout enforces it.

**Dirty-edit guard (AC-3.5).** When any cell is dirty (typed but unsaved), the sidebar shows a tooltip on hover: *"Save or discard your changes before switching languages."* Sidebar entries become `aria-disabled="true"` with `cursor-not-allowed`; tab swap is similarly blocked with a hint on the disabled tab.

### 3.3 — Create-version: right-edge drawer

```
                              ┌────────────────────────────────────────┐
                              │ Create version: Marketing campaigns ✕  │
                              │ Target language: Italiano (it)         │
                              │                                        │
                              │ ┌────────────────────────────────────┐ │
                              │ │  ○ Empty start                     │ │
                              │ │      Start with no mappings        │ │
                              │ ├────────────────────────────────────┤ │
                              │ │  ● Copy from another language      │ │
                              │ │      ┌─ source language ─────────┐  │ │
                              │ │      │ Deutsch (de) — 12 mappings▾  │ │
                              │ │      └─────────────────────────────┘  │ │
                              │ │      Mappings will copy verbatim.   │ │
                              │ │      Shared settings (type, flags)  │ │
                              │ │      already apply to all languages.│ │
                              │ └────────────────────────────────────┘ │
                              │                                        │
                              │                       [Cancel][Create] │
                              └────────────────────────────────────────┘
        ← workspace stays                           ← drawer 40% width
          visible but dimmed                          slides in from right
          (scrim)
```

**Why drawer not modal.** The drawer keeps the list and matrix visible behind a `bg-background/40` scrim. The operator can still see what their language column looks like across maps while the drawer is open — informational reassurance. A modal would lose that context. Trade-off: drawer is narrower than a centered modal, so the source-language picker has to live in a tight column rather than a 2-col layout — accepted as the price of context preservation.

**Drawer states:**
- **default** — both options visible. Top option pre-selected? **No.** Following PRD-000's strict-no-default culture from ADR-0018, the radios start with neither chosen; the "Create" button stays disabled until one is selected. (This is a smaller stakes decision than delete, so we could pre-select "Empty start" — but consistency with the delete strict-no-default pattern keeps the muscle memory aligned. Operator types in the source-language picker only after choosing "Copy from".)
- **copy-source-empty** — when no other language has non-empty mappings, the "Copy from" radio is `aria-disabled="true"` with tooltip *"No other language has mappings to copy from."* — falls back to "Empty start" as the only enabled option, but still requires explicit selection.
- **loading** — Create button shows `@blok/spinner` inline; both radios + source picker disabled.
- **partial-failure** (ADR-0021) — drawer remains open with an inline `@blok/alert variant="warning"`: *"Partially populated version detected. The empty version was created but mappings did not copy. [Rollback] [Keep empty version]"*

**Mobile / narrow-Full-Page collapse.** Below 900 px viewport, the drawer becomes full-width (`w-full`) and slides from bottom (`side="bottom"`). Same content, same states.

### 3.4 — Delete: consequence-preview panel

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Delete redirect map                                                    │
│  Marketing campaigns                                                    │
│                                                                         │
│  Choose what to delete:                                                 │
│                                                                         │
│  ○ Delete current language version only — "Deutsch (de)"                │
│  ○ Delete entire map and all language versions                          │
│                                                                         │
│  ┌─ This will remove ─────────────────────────────────────────────┐     │  ← only renders
│  │ ✕ 6 mappings in `de`                                           │     │     after a radio
│  │ ✕ Display name "Marketing-Kampagnen" (de)                      │     │     is chosen
│  │                                                                │     │
│  └────────────────────────────────────────────────────────────────┘     │
│  ┌─ This will keep ───────────────────────────────────────────────┐     │
│  │ ✓ 12 mappings in `en`                                          │     │
│  │ ✓ 4 mappings in `ja`                                           │     │
│  │ ✓ Display names in `en`, `ja`                                  │     │
│  │ ✓ Redirect type, flags (shared — already per all languages)    │     │
│  └────────────────────────────────────────────────────────────────┘     │
│                                                                         │
│                                          [Cancel] [Delete this version] │
└─────────────────────────────────────────────────────────────────────────┘
```

**Strict no-default radios** (ADR-0018 carries). The consequence-preview is **hidden until a radio is chosen** — empty dialog body when both radios are unchosen, the action button is disabled. Only when the operator picks (a) or (b) does the consequence-preview render, populated with the exact data that will be touched.

**Visual grammar:**
- "This will remove" panel: `@blok/alert variant="destructive"` with `bg-destructive/10 border-destructive/30`. Each line uses `✕` (U+2715) monochrome glyph (per signature behavior — never `❌` codepoint) styled with `text-destructive`.
- "This will keep" panel: `@blok/alert variant="success"` with `bg-success/10 border-success/30`. Each line uses `✓` (U+2713) monochrome glyph styled with `text-success`.

**Delete-version-cut variant (R-1 cascade).** When `deleteItemVersion` is absent from the Authoring schema (PRD § 13 R-1 decision), only option (b) is visible. The dialog copy adjusts to: *"Single-version-delete is unavailable in this Authoring schema. Delete the entire map across all languages?"* — consequence-preview shows ALL mappings in ALL languages as "remove" (no "keep" section). Pre-selected? **Still no** (ADR-0018 holds even with one option) — the operator must click the radio to enable Delete.

**Single-version edge case (AC-8.6).** Same two-radio modal with an inline note below the radios: *"This map has only one language version; both options remove all redirect data. Choose explicitly to confirm intent."* Both radios start unselected (strict no-default holds).

### 3.5 — Import preview: grouped-by-language sections

```
┌─────────────────────────────────────────────────────────────────────────┐
│ Import preview — multilingual-bundle.json                               │
│ 12 maps × 3 languages (en, de, fr)                                       │
│                                                                         │
│ ▼ English (en) — 12 maps · 8 create · 3 overwrite · 1 skip              │
│   ┌──────────────────────────────────────────────────────────────┐      │
│   │ Marketing campaigns · 12 mappings · NEW                       │      │
│   │ Legacy URLs · 28 mappings · 2 changes · [overwrite ▾]         │      │
│   │ A/B variants · 4 mappings · IDENTICAL · [skip ▾]              │      │
│   │ …                                                             │      │

```

**Why grouped-by-language not grouped-by-map.** The operator's mental model when reviewing an import is "I want to see what's coming in for `de`" — they collapse the languages they're confident about and focus on the one they're vetting. This matches the export bundle's `languages` keyed structure (architecture § 4.5). The action picker stays at map level (FR-16) — the per-language sub-axis is OQ-6 deferred — so the dropdown appears once per map row, regardless of which language group renders it. (Each map appears in every language group it's present in; the action picker is bound at the map-row level via the map GUID, so changing it in one group reflects in the others.)

**Long-list relationship visibility (per signature behavior).** Each language group is collapsible (`@blok/accordion`). When collapsed, a one-line summary persists (`8 create · 3 overwrite · 1 skip`). When expanded, the table renders. At 30+ maps × 5 languages, the operator collapses 4 of 5 groups and works one language at a time — the relationship between map name and action stays on a single row inside each table.

### 3.8 — Critical user flows

**Flow A — Switch language, edit a redirect, save.**
1. Operator opens Full Page; sidebar shows last-used language selected (e.g. `de`); matrix list lights up the `de` column.
2. Operator clicks `fr` in sidebar → matrix recomputes; map detail clears (no map selected).
3. Operator picks "Legacy URLs" from list → detail pane renders with `fr` tab active.
4. Operator double-clicks a mapping row → cell becomes editable.
5. Types new target → `Save` button activates → click → toast: *"Saved to fr version."*
6. Sidebar `aria-current` stayed on `fr` throughout (no shift); tabs scrolled `fr` to the leftmost slot.

**Flow B — Create a `fr` version by copying from `de`.**
1. Operator in sidebar on `fr`; matrix list shows `○` in `fr` column for "Marketing campaigns".
2. Clicks the `○` cell → detail pane opens with "Marketing campaigns" + tab strip showing `en` `de` `fr (create)` `ja`.
3. Operator clicks `fr (create)` tab → drawer slides in from right with target `fr` pre-filled.
4. Picks "Copy from another language" → source picker reveals → picks `de — 12 mappings`.
5. Clicks `Create` → spinner → drawer closes → toast: *"Copied 12 mappings from de to fr."*; tab strip now shows `fr*` (dirty? No — newly created and committed) → `fr` becomes the active tab; mappings list pre-populated.

**Flow C — Delete `de` version only.**
1. Operator viewing "Marketing campaigns" with `de` tab active.
2. Clicks `Delete` in topbar → consequence-preview dialog opens with both radios unselected; consequence panels hidden.
3. Operator picks "Delete current language version only — `de`" → consequence panels populate live: 6 mappings + display name removed; 12 + 4 mappings preserved.
4. Clicks `Delete this version` → mutation runs → toast: *"Deleted de version of Marketing campaigns."*; tab strip recomputes (`de` becomes `de (create)`); detail pane stays on map with sidebar still on `de`.

**Flow D — Edit `__Display name` for `fr`.**
1. Operator on `fr` tab of a map whose `fr` `__Display name` is empty; title renders italic placeholder.
2. Clicks the placeholder → title becomes input (`@blok/editable`); pre-filled empty; cursor at start.
3. Types "Campagnes marketing" → presses `Enter` → mutation runs → title commits to non-italic display name; toast confirms.
4. (Alternative path) Types something, presses `Escape` → input reverts to placeholder; nothing written; no toast.

**Flow E — Import a multilingual bundle.**
1. Operator clicks `Import` in topbar → file picker → selects v2 JSON.
2. Validator runs → bundle accepted → preview screen renders with three language groups; `en` group expanded by default (first in `siteLanguages` order).
3. Operator collapses `en`, expands `de` → reviews → picks "overwrite" on three maps that show diffs.
4. Clicks `Apply 12 changes` → progress UI → summary screen: per-map outcome with per-language detail.

**Flow F — Open the Context Panel on a `de` page (no behavior change).**
1. Operator in Sitecore Pages on a `/de` URL → opens Page redirects panel.
2. Panel renders with workspace-style banner naming the `de` page; matcher still `en`-only → only `en` redirects shown.
3. Operator clicks one → Sitecore Authoring navigates (no PRD-001 change).

### 3.9 — Responsiveness summary

See § 5.1 for the breakpoint table. Critical V3 collapse points:

- **Sidebar → top-bar `@blok/select`** at < 900 px viewport. The select shows the same languages; choosing one updates picker state same as the sidebar.
- **Tabs → horizontally scrollable strip** at < 700 px detail-pane width (with scroll buttons rendered via `@blok/scroll-area`).
- **Drawer → bottom-edge sheet** at < 900 px viewport.
- **Matrix columns → collapsed badge** at < 600 px viewport: the per-language dot row is replaced by a single `@blok/badge` showing "3 of 5 languages" — operator taps to expand inline.

### 4.3 — Accessibility notes (per Blok pattern)

**Sidebar (`@blok/navigation-stack`):**
- Root `<nav aria-label="Languages">`.
- Each row is `<button>` with `aria-current="page"` on the active one. Roles `listbox` + `option` rejected because the sidebar is navigation, not single-select-from-list semantically.
- Keyboard: `↑` / `↓` move focus; `Home` / `End` jump; `Enter` / `Space` activate.
- Focus ring is `ring-2 ring-ring ring-offset-2` (structural focus-visible guard from PRD-000 carries — every interactive element must pair `outline:none` with a `:focus-visible` style).
- Screen-reader announces row label, ISO code, and "current" state via `aria-current`.

**Tabs (`@blok/tabs`):**
- Standard ARIA tabs pattern. Tab list has `role="tablist"`; each tab `role="tab"` + `aria-selected`; panel `role="tabpanel"` + `aria-labelledby`.
- Missing-version tab carries `data-state="missing"` + `aria-label="Create [language] version"` so SR distinguishes it from existing tabs.
- Keyboard: `←` / `→` move focus between tabs (Arrow keys, not Tab); `Enter` / `Space` activate; `Home` / `End` jump.

**Drawer (`@blok/sheet`):**
- `role="dialog"` + `aria-modal="true"` + `aria-labelledby` pointing to the drawer title.
- Focus trap: initial focus on first radio; `Tab` cycles through interactive elements in the drawer only; `Escape` closes (asks for confirm if dirty).
- Returns focus to the trigger element on close (the cell or tab that opened it).

**Consequence-preview dialog (`@blok/alert-dialog`):**
- `role="alertdialog"` + `aria-modal="true"` + `aria-describedby` pointing to the radios container (NOT the consequence panels, because they're hidden until a radio is chosen — SR reads the radios first, then on selection the live region announces the consequence summary).
- Live region (`aria-live="polite"`) wraps the consequence panels so newly-revealed content announces.

**Click-to-edit title (`@blok/editable`):**
- Idle state: title is `<h1>`; clicking dispatches the editable activation.
- Editing state: input has `aria-label="Display name for [language]"`. `Enter` commits, `Escape` cancels. Blur does NOT commit (AC-9.4) — explicit Save button or Enter required.
- Placeholder text rendered with `aria-label="No display name set for [language] — click to add one"` so SR identifies the affordance.

**Coverage matrix cells:**
- Each cell is a `<button>` with `aria-label="[map name] · [language]: [version exists with N mappings | version exists but empty | no version — click to create]"`.
- Focus moves cell-by-cell via Tab; `Enter` activates (opens detail with that tab pre-selected, or opens the create-version drawer for `○` cells).
- Color is paired with shape (filled dot / half dot / outline-only) so colorblind operators don't lose the signal.

**Color contrast (NFR-A4 + WCAG 2.1 AA):**
- Active sidebar row: `text-foreground` on `bg-accent/40` — verified ≥4.5:1 in both light + dark theme via Blok semantic-token defaults.
- Coverage badges: `bg-success` dot on `bg-card` row — Blok success token has built-in contrast pairing.
- Italic placeholder display name: `text-muted-foreground italic` — verified ≥3:1 (large text threshold; ≥18 px due to `text-2xl`).
- Consequence panels: `text-destructive` on `bg-destructive/10` and `text-success` on `bg-success/10` — Blok semantic pairings; verified.

## 5. Responsiveness and accessibility

### 5.1 — Breakpoints

PRD-001 inherits the iframe viewport budgets from `blok-marketplace-integration` skill. V3 collapse map:

| Viewport (Full Page iframe width) | Sidebar | Detail tabs | Drawer | Matrix |
|---|---|---|---|---|
| ≥ 1280 px | sticky 220 px | full row, all tabs visible up to 8 | right-edge 40% width | full per-language columns |
| 1024–1279 px | sticky 220 px | row scrolls horizontally above 6 tabs | right-edge 50% width | full columns; row meta compacts |
| 900–1023 px | sticky 200 px | row scrolls; scroll-button affordances visible | right-edge 60% width | columns kept; row gap reduces |
| 600–899 px | **collapses to top-bar `@blok/select`** | scroll-only with focus-visible arrows | bottom-edge sheet, full width | full columns or single "N of M languages" badge below 700 px |
| < 600 px | top-bar select | scrolling tabs | bottom sheet | collapsed badge |

Context Panel (320–400 px) and Dashboard (300–800 px) never receive the sidebar; their layouts are documented in § 3.6–3.7 and don't change responsively beyond standard Blok primitives.

### 5.2 — Tab horizontal scroll behavior

When the tab strip overflows its container, `@blok/scroll-area` wraps it horizontally with edge-fading affordances. `←` and `→` keys still navigate between tabs even when offscreen — the active tab auto-scrolls into view. Touch swipe is supported (Blok primitive default).

### 5.3 — Sidebar collapse to select

At < 900 px viewport: the sidebar `<nav>` hides; in its place, the topbar grows a `<label>Language</label><select>` that lists all enumerated languages with the active language pre-selected. Choosing a different option fires the same `setLanguage` mutation as a sidebar click. The select gets `aria-label="Language"` per AC-1.x carryover. On viewport widening back to ≥ 900 px, the sidebar re-renders; the select disappears; picker state is preserved (single source of truth in `localStorage`).

### 5.4 — WCAG 2.1 AA target

- **1.4.3 Contrast (Minimum):** all Blok semantic pairings used by V3 satisfy ≥4.5:1 for normal text, ≥3:1 for large text. New color pairings (consequence panels) verified above.
- **1.4.11 Non-text Contrast:** sidebar accent bar, focus ring, coverage dot — all ≥3:1 vs adjacent.
- **1.4.13 Content on Hover:** the matrix-cell tooltips on hover are dismissible (cursor leave dismisses; `Escape` dismisses for keyboard-triggered focus), hoverable, and persistent (don't auto-dismiss).
- **2.1.1 Keyboard:** every interactive surface in § 4.3 is keyboard-operable. No mouse-only paths.
- **2.1.2 No Keyboard Trap:** drawer + dialogs use Blok primitives' default focus trap with `Escape` exit.
- **2.4.7 Focus Visible:** structural guard from PRD-000 enforces — every `outline:none` paired with a `:focus-visible` style. V3 specifies the ring as `ring-2 ring-ring ring-offset-2` everywhere.
- **3.3.3 Error Suggestion:** form errors (drawer, click-to-edit, save) include suggestion text (e.g. *"Display name must be 1–255 characters."*).
- **4.1.3 Status Messages:** toasts use `@blok/sonner` which announces via `role="status"` (success/info) or `role="alert"` (errors).

### 5.5 — Reduced motion

`prefers-reduced-motion: reduce` disables drawer slide animation (drawer appears instantly), accordion expand/collapse animation (state changes instantly), and matrix cell tooltip fade (instant show/hide). No content is hidden purely by motion — the drawer always remains positioned right-edge with the same content.

### 5.6 — RTL

Blok's RTL support carries (per `blok-theming` skill). The sidebar moves to the right edge under `dir="rtl"`; the drawer slides in from the left edge. Tabs read right-to-left. The PRD-001 release language set is `en` / `de` / `fr` / `it` / `ja` (LTR languages) — RTL is not a blocking requirement for ship but the design must not preclude future addition of Arabic / Hebrew sites.
