## 1. Summary

PRD-003 wires the placeholder "Publish all" hero CTA on the Full Page workspace to a real **SitecoreAI Publishing v1** publish job (renamed to "Publish Site"), adds **lightweight cross-session job-status tracking** (3-second polling + Tier 2 localStorage resume + Tier 3 name-prefix list scan), and introduces the first **server-side OAuth proxy route** in Redirect Manager (Branch B per ADR-0034 — narrowly carved out of ADR-0002 by ADR-0035). Two operator findings during real-tenant smoke triggered a Tranche 3 amendment: **per-map publish removed** (Sitecore silently no-ops Items publish for Redirect Map items — not Edge-published content) and **locale resolver simplified to `["en"]` only** (UrlMapping is SHARED across languages, making multi-locale publish redundant). Final shipped surface: rename + Site publish + status visibility, on the Full Page extension point only. Context Panel and Dashboard Widget unchanged.

## 4. Known Limitations

- **Single language publish (`["en"]` only)** — by design, because `UrlMapping` is a SHARED Sitecore field (no language axis on Redirect Map items). Operators who configure other languages on their site will see their redirect data still propagate to Edge; the `en` publish covers all language versions because the field is SHARED. Documented in `lib/publish/locale-resolver.ts` header comment + ADR-0036.
- **Operator must register a Cloud Portal automation client** with scopes `xmcloud.cm:admin` + `xmcpub.jobs.a:r` + `xmcpub.jobs.a:w`. Documented in `.env.example` + `site/docs/smoke-publish.md`. This is the per-tenant setup step required by Branch B / ADR-0035.
- **Host-frame Playwright pixel comparison was skipped** — no POC was produced this PRD (light-rigor scope dial). Visual fidelity verified by operator live walkthrough during m_publish smoke. Acceptable per scope_dial; recorded in `smoke_outcomes.host_frame_smoke: skipped`.
- **Cross-machine resume is best-effort** — Tier 3 list-scan filters to jobs from the last 60 minutes only. A publish started >60 min ago on another machine is treated as terminated; the user can verify via SitecoreAI's publishing list directly. Acceptable for the "lightweight" charter.
- **Polling cadence is 3s with no progress UI** — operators see "Publishing… Xs" elapsed display on the button, but no per-item progress bar. Statistics are available via the API (`statistics.itemsProcessed` / `itemsSent`) and could be surfaced in a future PRD if needed.

## 6. Technical Debt Introduced

- **`locale-resolver.ts` accepts `site` + `shorthandAccepted` args it ignores** — function intentionally retains API stability so future multi-locale work doesn't ripple through callers. Header comment documents the deliberate trade-off; tests assert the simplified contract. Low-cost; revisit if a future PRD reintroduces multi-locale publish.
- **`config.ts` exports `PUBLISH_LOCALE_SHORTHAND_ACCEPTED: false`** — currently unread (resolver returns `["en"]` regardless). Kept for symmetry with the Tranche 1 D-T1.2 probe pattern; can be deleted on a future cleanup pass.
- **Sonner adapter `dismissId` is unused for terminal toasts** — Tranche 3 introduced the stable `publish-job-<jobId>` pattern as a separate toast lifecycle managed by `WorkspaceHero` directly. The original adapter's `dismissId` parameter still works (used by `publish()` orchestration's `queued()` call) but the polling terminal path bypasses it. Two parallel lifecycles; could consolidate in a future cleanup.
- **Server-side body validation defense gap** (deferred — see § 3).
- **`jest-axe` not installed** — listed in § 5 open test gaps.
