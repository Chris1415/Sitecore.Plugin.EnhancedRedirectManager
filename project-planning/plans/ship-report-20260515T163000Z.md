## 6. Technical Debt Introduced

- **Skeleton-on-refresh** was reimplemented three times in this PRD (dashboard widget, Full Page rail, Context Panel) before settling on the "prior-data short-circuit" pattern. Worth promoting to a framework rule (`async UI: skeleton only when priorData.length === 0`).
- **`.elev-modal-content`** is referenced by 3 modal components but not defined as a CSS rule. It works only because undefined Tailwind classes are no-ops. Should either be removed or formalized as a CSS rule with intent.
- **`PREVIEW_DATA_ACTIVE` flags** are partially obsolete after this polish wave wired real data into the Dashboard Widget and Full Page stat strips. The flags + module should be retired or repurposed.
- **`HealthBadge` mock placeholder** ships as user-visible UI without a real probe behind it; risk that an operator reads "all healthy" as a real verdict.
- **`AddRedirectModal` already removed** (ADR-0028 Option A) — no debt here, called out for completeness.

## 8. Recommendations for Next Iteration

1. **PRD-003 candidate — data plumbing wrap-up.** Wire HealthBadge to a real probe worker; promote Validate health + Publish all from decorative to real; retire PREVIEW_DATA_ACTIVE flags.
2. **Framework rule — async UI skeleton policy.** Add a rule that loading skeletons only render when `priorData.length === 0`; subsequent refreshes keep content visible and use trigger-side indicators (spinning button icon). Fixed three times this PRD.
3. **Framework guard — `.elev-modal` vs `DialogContent`.** Either rename to `.elev-modal-backdrop` or add a structural guard banning `.elev-modal` on any `<DialogContent>`. The naming collision broke the Edit map dialog and was caught only by screenshot review.
4. **`useDirectionToggle()` hook.** Both Conflicts → Open map and QuickRedirectForm direction toggle implement the same flip-state-with-label-swap pattern. Worth extracting.
5. **`validateMapMutation()` extractor.** Four write paths (Full Page add, Full Page edit, Context Panel quick-add, Context Panel inline edit) need the same in-map duplicate-source guard. Three had it; the inline edit gap was caught only during the polish wave. Worth a single pure function.
6. **Playwright per-theme visual smoke.** Catches token-vs-theme contrast issues (low-contrast tints in dark mode) before they reach the operator. Add to QA's smoke checklist.
7. **POC clickdummy parity discipline.** When operator polish lands a UX change, the POC clickdummy should be updated in the same loop. Otherwise the POC becomes a stale source of truth for the next product.
