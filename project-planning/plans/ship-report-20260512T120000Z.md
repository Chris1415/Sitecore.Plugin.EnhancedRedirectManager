## 1. Summary

PRD-000 ships **Redirect Manager** — a Sitecore Marketplace client-side app (Mode A) with three Cloud Portal extension points (Context Panel, Dashboard Widget, Full Page) that performs Redirect Map item CRUD via Sitecore Authoring GraphQL on the `en` language version, plus JSON import / export keyed by Sitecore item GUID. The MVP scope (en-only, pure CRUD, no analytics, no concurrent-edit detection) was delivered end-to-end across nine implementation tranches; all five real-tenant smoke gates passed against `CHAH DevEx Journey / PROD`. Heavy documentation pass (`docs/architecture.md`, `docs/decisions.md`, README with feature gallery, CHANGELOG) ran on demand per `scope_dial.docs_at_ship: full`.

## 4. Known Limitations

Acknowledged caveats — none rise to "broken" but reviewers should know:

- **Cloud Portal does not pass per-site context to Dashboard Widget embeds today.** The widget's `ApplicationContext` is tenant-scoped only; the widget falls back to a site picker (operator's last choice persisted via `localStorage`). The host-frame `document.referrer` auto-detect is present but has not been observed to fire usefully — picker is the honest UX. Documented in README and in `docs/architecture.md` § 2.
- **`Redirect307` is rejected by the head-app resolver.** Server accepts the value at the field level but the resolver does not honor it. Removed from the UI picker; documented in `docs/decisions.md`.
- **Import on cross-tenant promotion mints fresh GUIDs** for `create` actions. `CreateItemInput` does not accept a caller-supplied `id` (ADR-0009 closed with definitive NO during Tranche 6a). The import summary surfaces "newly-minted GUID" per item.
- **`__Updated` field uses Sitecore compact format (`yyyyMMddTHHmmssZ`), not ISO-8601.** `parseSitecoreCompactDate` handles both. Recorded so future SDK migrations don't silently regress.
- **Host-frame visual smoke is operator-driven and documented, not automated.** T064 was deferred from a Playwright harness to a documented manual procedure in `site/docs/host-frame-smoke.md`. The 5-axis comparison can be automated later by following the recipe and pointing at the POC clickdummy.
- **Marketplace SDK envelope quirks** for `xmc.authoring.graphql` mutations (body inside `params`, double `.data.data` unwrap) — captured inline in `lib/sdk/redirects-write.ts` + decisions doc. Wrong sibling-app patterns existed and were pushed back to the `sitecore:marketplace-sdk-{client,xmc}` skill docs.
