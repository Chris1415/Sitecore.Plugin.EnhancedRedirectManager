## 1. Design context & assumptions

- **Audience.** Three operator personas carry from PRD-002 § 4. Marketers (most frequent — daily Context Panel use; want the fastest add-redirect path). Site administrators (Full Page workspace daily; CRUD on Redirect Maps + mappings; export/import). Migration operators (Full Page import wizard; one-shot intensive sessions). Stakeholders glancing at the Dashboard Widget tile in the Cloud Portal home (marketing-grade first impression matters most here). All operators interact through the Cloud Portal iframe — never standalone — at a desk, evergreen Chromium / Firefox / Safari, light AND dark theme.

- **Goals and success criteria.**
  1. Every redesigned surface reads as **marketing-grade premium** while staying entirely inside the Blok Nova design system (no off-brand fonts, no invented hex, no non-Blok component primitives).
  2. The Context Panel quick-add task drops from 4 clicks (button → modal → fill → save) to 2 clicks (type into always-visible inline form → Add) per ADR-0028 + AC-R5.
  3. Mock content reads as honest preview, not as live data — single per-surface "Preview data" banner pairs with `data-preview-mock="true"` annotations (FR-R3).
  4. Mixed motion budget per ADR-0027 — full V4 motion on Full Page (drifting plumes, kinetic letter reveals, count-ups on the 4 stat tiles); quieter on Context Panel + Dashboard Widget (hover lifts + a single hero count-up only).
  5. Every animation respects `prefers-reduced-motion: reduce`. Static fallbacks remain visually pleasant — gradients persist, only the loops stop.
  6. WCAG 2.1 AA across light + dark mode. Focus-visible carries from PRD-000.
  7. The refined POC is the canonical visual source of truth for implementation. When the spec text and the POC diverge on a visual detail, the POC wins (per glossary § POC clickdummy).

- **Constraints.**
  - **Stack:** Next.js 16 SPA inside Cloud Portal iframe; Blok via shadcn registry (already installed in PRD-000); Tailwind v4 Nova preset (no `tailwind.config.ts`); React 19; `@sitecore-marketplace-sdk/client` + `/xmc`; existing token surface at `site/app/globals.css`.
  - **Typography:** Geist Sans + Geist Mono via `next/font/google` (PRD-000 `app/layout.tsx:8-16`). No Inter, no Space Grotesk, no Fraunces, no editorial serif. Hero clamp range 48-96px on Geist Sans 700 (per architecture § 9 R-13 + PRD § 11 Design contract values).
  - **Color discipline:** Zero invented hex values outside `site/app/globals.css`. Every color expression composes Blok semantic tokens through `color-mix(in oklch, ...)`. The `@blok/badge`, `@blok/button`, `@blok/card`, `@blok/dialog`, `@blok/input`, `@blok/select`, `@blok/alert`, `@blok/sonner` primitives — all theme-aware.
  - **Viewport / extension-point constraints** (per `sitecore:blok-marketplace-integration` § "Iframe viewport constraints"):
    - Full Screen iframe (`/full-page` route): 1024-1920px wide, 600-1080px tall.
    - Pages Context Panel iframe (`/context-panel` route): 320-400px wide, variable height + scrollable. **Use 360px as primary design width per V4 POC reference.**
    - Dashboard Block iframe (`/dashboard-widget` route): 300-800px wide, 200-400px tall. **Use 480px as primary design width per V4 POC reference.** Note: per Architecture Decision 1, the widget renders at compact size and is one tile among many.
  - **Accessibility:** WCAG 2.1 AA carries from PRD-000 NFR-A1..NFR-A5. Focus-visible 2px solid `var(--ring)` + 2px offset (theme.css `:focus-visible`). Color contrast verified in both light and dark modes (dark-mode `--primary-foreground` override locked in PRD-000 to fix Nova-preset collapse). All decorative motion + decorative SVG `aria-hidden`.
  - **No new dependencies.** No `framer-motion`, no `react-spring`, no `motion-one`. Vanilla CSS @keyframes + ~30 LOC vanilla JS for letter-split and count-up (architecture § 7 Technology Stack).
  - **HahnSoloFooter carries** at `position: fixed; bottom: 0; right: 0; z-index: 50; pointer-events: none` — must remain visible on every redesigned surface. Plume backdrop sits at `z-index: -1` (architecture Decision 4 z-index hierarchy).

- **References / inspiration.**
  - **V4 Blok Elevated marketing POC** at `pocs/poc-marketing-v4-blok-elevated/` — the canonical base for this refinement. Selected by operator from 4-variant marketing exploration (2026-05-13). README rationale: *"unmistakably Blok, dressed for a marketing page."*
  - **Blok design system** docs at `https://blok.sitecore.com/primitives` + `https://blok.sitecore.com/bloks` + `https://blok.sitecore.com/theming` — single source of truth for component anatomy + tokens.
  - **PRD-000 V1 POC** at `pocs/poc-v1/` — utility-tool visual baseline that PRD-002 elevates above. Carries the working layout grammar (left rail + map detail; matched-row anatomy; Dashboard 3-tile pattern); PRD-002 keeps that grammar but lifts every surface visually.
  - **Cloud Portal Blok-installed sibling app** at `products/last-edit-trail/site/app/globals.css` — canonical Blok Nova token source already copied verbatim into V4's `theme.css`.

- **Answers to clarifying questions** (PRD § 11 Open at `/architect` + architecture § 11.1):
  - *OQ-1 marketing copy.* Use V4's anchor copy with reality-validated edits: Full Page hero headline `"Eight active maps, all healthy."` (`Eight` = real-data `activeMapsCount`; `"all healthy"` = mocked); Dashboard Widget headline = `"Your redirect operations, at a glance."`; Context Panel hero header = `"N redirects point here."` (`N` = `matchedGroups.length`). All three may be operator-overridden during `/architect` review.
  - *OQ-2 Preview Data banner copy.* `"Some metrics on this surface use preview data — wired up in a follow-on release."` Plain Blok `--info`-tinted `@blok/alert` anatomy with a small "info" glyph (monochrome `i`, not the emoji `ⓘ`).
  - *OQ-9 Inline form map-selection model.* Architecture Decision 2 + ADR-0029 locked: most-recently-updated default; multi-match affordance exists; **POC ships a dropdown** (compact, accessible inside a 360px panel — chip cluster gets cramped with 3+ map names).
  - *OQ-10 Hero CTA wiring.* Architecture Decision 3 + ADR-0030 locked: decorative; onClick → Sonner toast.
  - *Multi-match affordance — chip cluster vs dropdown.* **Committed: dropdown** (the `@blok/select` primitive). Rationale: at 360px Context Panel width, chip clusters wrap into 2-3 rows for typical map names (`"Legacy WP Imports"`, `"Black Friday 2025"`); dropdown stays one row + collapses long names with `text-overflow: ellipsis`.

- **Assumptions made (carry to handoff).**
  1. The 3 real Dashboard Widget tiles (Maps / Mappings / Last-updated) inherit V4 chrome (frosted card surface + tabular-num typography + V4 hover lift) — they are NOT mocks. Data still flows through `aggregateStats` (PRD-002 § 5).
  2. `RedirectType` is rendered as 3 values everywhere: `301` / `302` / `Server Transfer` (display labels via `redirectTypeDisplayName()` per `site/lib/redirects/redirect-type-enum.ts`). No `307` literal anywhere in the design.
  3. The Context Panel matched-row group affinity (which redirect belongs to which parent map) is shown via a **parent-map name + last-updated meta line per row** — same pattern PRD-000 already ships. This satisfies the "long-list relationship-visibility check" because the parent-map context is in-row, not in a parallel column.
  4. The "+ New map" affordance from the inline form (when ADR-0029 needs the create-new path) opens NOT a modal — there's no modal — but transitions the form into create-new state inline (RedirectType select enables; an auto-name preview shows beneath the source input). This is the simplest UX and matches ADR-0028's "no AddRedirectModal" commitment.
  5. Motion gating applies via `@media (prefers-reduced-motion: reduce)` AND via JS `window.matchMedia('(prefers-reduced-motion: reduce)').matches` early-return in the count-up helper (per V4 `script.js` line 10-12).

## 2. Design direction

**Named direction — "Blok Elevated" (V4-refined).** Premium-grade composition INSIDE the Blok system. Same Geist Sans / Geist Mono. Same lavender `--primary` (`#6e3fff` light / `#d9d4ff` dark). Same `--card`, `--muted`, `--border`. But higher composition ambition: hero typography on the Blok type scale at clamp(48px, 8vw, 96px), gradient-clip text composed from `linear-gradient(135deg, var(--primary), color-mix(in oklch, var(--primary) 55%, var(--info)))`, frosted-glass surfaces over a slow-drifting `--primary` / `--info` plume backdrop (Full Page only), primary-tinted hover glows (`box-shadow: 0 12px 32px color-mix(in oklch, var(--primary) 14%, transparent)`), kinetic letter reveals on the workspace headline, count-up animation on the stat strip's 4 numbers (Full Page) and on the hero stat (Dashboard Widget).

**The aesthetic move:** take the operator-grade Blok palette (lavender primary, neutral grays, paper-white card) and apply marketing-grade composition. Generous whitespace at hero scale. `cubic-bezier(0.16, 1, 0.3, 1)` easing throughout (the "Blok premium ease"). Every elevation step composes Blok semantic tokens via `color-mix(in oklch, ...)` — no invented hex, no off-Blok typefaces, no custom palettes. **It is unmistakably Blok, dressed for a marketing page.**

**Voice.** Hybrid per PRD-002 D5 (revised):
- **Marketing voice** in three explicitly designated zones:
  1. **Full Page workspace hero headline** — `"Eight active maps, all healthy."` (gradient-clip on `"all healthy."`).
  2. **Dashboard Widget headline** — `"Your redirect operations, at a glance."`.
  3. **Context Panel hero `<h1>` count header** — `"N redirects point here."` (gradient-clip on the count token).
- **Utility-tool voice** everywhere else: modal titles ("Create new Redirect Map", "Delete map", "Import redirects"), form field labels, table column headers, button labels, validation messages, footer text. D5 explicitly does NOT extend to modal titles — V4's brief modal titles read as utility labels.

**Layout and structure (per surface).**

- **Full Page (`/full-page`)** — Roomy three-band shell at 1024-1920px wide:
  - Sticky frosted topbar (`z-index: 40`, `backdrop-filter: blur(18px) saturate(160%)`) with breadcrumb + workspace title + Import/Export/New-map actions.
  - **Preview Data banner** mounted directly under the topbar (single `@blok/alert` with `--info` tint).
  - **Workspace hero zone** (NEW per ADR-0024): eyebrow chip (`"Workspace · MainSite EN"`), gradient-clip hero headline (Geist Sans 700 clamp(48px, 8vw, 96px)) with kinetic letter reveal on first paint, sub-line (`"Last publish 14 minutes ago by Anna. No conflicting source URLs detected."` — `"Edge caches refreshed across 7 languages"` REMOVED per FR-R11), two decorative CTAs (`View activity` + `Publish all` — toast onClick per ADR-0030).
  - **Stat strip** (4 frosted tiles) above the map detail — each tile shows a count-up number (`68` Mappings / `64` 301 / `4` 302 / `0` Conflicts — all from `PREVIEW_DATA.fullPageStatStrip`), an uppercase label, a small sub-line, and a subtle primary-tinted gradient spark at the bottom. `data-preview-mock="true"` per tile.
  - **Two-column body** at ≥1024px: left rail (sticky `Redirect maps` list at 320px) + right main column (map detail card with header + tools row + table). Below 1024px, rail collapses above the main column.
  - **Drifting plume backdrop** (`.fp-plume-backdrop` at `z-index: -1`) renders 2 large blurred radial-gradient discs that translate slowly via the 28s `backdrop-drift` keyframe. `prefers-reduced-motion: reduce` halts the loop.
  - **HahnSoloFooter** at `z-index: 50` bottom-right corner.

- **Context Panel (`/context-panel`)** — Compact 360px panel:
  - No drifting plumes (motion budget per ADR-0027). Static `color-mix` radial wash on body background.
  - **Hero `<h1>` count header** (NEW per ADR-0024 + D5 zone c): eyebrow `"Redirects for this page"` + `<h1>` `"N redirects point here."` with gradient-clip on the count number + page route in mono font beneath. ~36-44px clamp (smaller hero than Full Page — fits 360px width).
  - **Hero summary tile** beneath the header: `"Sources redirecting to this URL"` + repeated count (count-up animation on first paint).
  - **Inline `QuickRedirectForm`** card (NEW per ADR-0026 + ADR-0028): always visible. Source input pre-populated with current page route (mono font) + RedirectType select (3 values: `301`, `302`, `Server Transfer`) + `→ this page` affordance + Add button. The form's RedirectType select is enabled in the no-match (create-new) state and disabled-with-display-only in the match (add-to-existing) state per ADR-0029 — small "uses existing map's redirect type" hint beneath.
  - **Multi-match dropdown** (POC commitment per OQ-9): when `matchedGroups.length >= 2`, a labeled dropdown (`@blok/select`) renders above the inline form's source input, reading `"Adding to: <map-name>"` with a chevron and list of all matched maps sorted by `updatedAt` descending. Selecting a different map re-binds the form's disabled RedirectType select.
  - **Matched redirect rows** beneath the form: each row is a `@blok/card` with source → target (mono) + `@blok/badge` for RedirectType (one of `301` / `302` / `Server Transfer`) + parent-map name + last-updated meta. **No status pill. No `Active` / `Draft` label. No `--draft` dot variant. No `"unpublished"` meta.**
  - **RegexBanner** (carry from PRD-000) re-skinned per `@blok/alert--info` chrome.
  - **"Open full workspace" CTA** at the bottom routes to `/full-page`.
  - **HahnSoloFooter** at `z-index: 50` (visible inside panel iframe but may clip — same as PRD-000).

- **Dashboard Widget (`/dashboard-widget`)** — Compact 480px tile:
  - **Preview Data banner** at the top (single `@blok/alert` with `--info` tint).
  - **Widget header** with `<h2>` `"Redirect Manager"` (utility — NOT marketing zone), sub-meta `"SneakerCo · MainSite · EN"` (real data from `application.context`), and `Open` ghost button → `/full-page`.
  - **Marketing-grade hero subhead**: `"Your redirect operations, at a glance."` (Geist Sans 600 ~16-18px — fits the marketing voice zone in compact form per D5 zone b).
  - **Hero stat zone**: huge `12,428` count-up number (Geist Sans 700 ~72px, gradient-clip with `--foreground` → `color-mix(--primary 70%, --foreground)`) + sub-stack (`"Active redirects"` uppercase mono label + `"+412 this week"` success-foreground delta). All from `PREVIEW_DATA.heroStat`. `data-preview-mock="true"`.
  - **Gradient sparkline** (21 mock points from `PREVIEW_DATA.sparkline`, fill gradient `currentColor` top → transparent bottom, stroke at `var(--primary)`). `aria-hidden="true"`.
  - **3 real tiles** (Maps / Mappings / Last-updated) — V4 chrome (frosted card surface, tabular-num typography) but NOT mocks. Data from `aggregateStats`. NO `data-preview-mock` attribute.
  - **4th tile** (mock) — `"Recently shipped"` count with `"+4 last 24h"` sub. From `PREVIEW_DATA.recentlyShipped.countLast24h`. `data-preview-mock="true"`.
  - **Top destinations** (5 rows, mock) — each row: target path (mono, en-only paths: `/products`, `/black-friday`, `/campaigns/promo`, `/sale`, `/early-access`) + gradient bar fill + hit count (mono tabular-num).   - **Recently shipped mini-widget** (3 rows, mock) — source → target (mono ellipsis) + RedirectType badge. NO status pill. From `PREVIEW_DATA.recentlyShipped.rows`. All paths en-only. `data-preview-mock="true"`.
  - **Footer attribution** (mock) — `"Last publish 14m ago by Anna"` + `"all healthy"` success badge (monochrome check glyph, NOT emoji ✅). From `PREVIEW_DATA.dashboardFooter`. `data-preview-mock="true"`. **The PRD-000 `FootnoteSeparated` `"Redirect counts only — usage analytics ship in a follow-on release"` line is REMOVED** (consolidated into the banner per AC-R3.7).

## 3. Key screens and flows

### 3.1 Surface inventory

| Surface | POC file | State | Width baseline |
|---|---|---|---|
| Landing / navigation index | `index.html` | static — entry point to the POC frames | adaptive (max 1200px) |
| Full Page workspace | `full-page.html` | default — map list + selected map detail + stat strip | 1280px design |
| Context Panel — single match (common case) | `context-panel.html` | `matchedGroups.length === 1` | 360px |
| Context Panel — multi-match | `context-panel-multi-match.html` | `matchedGroups.length === 3` (3 matched maps) | 360px |
| Context Panel — no match (create new) | `context-panel-no-match.html` | `matchedGroups.length === 0` | 360px |
| Dashboard Widget | `dashboard-widget.html` | default — hero + sparkline + tiles + top destinations + recently shipped | 480px |

### 3.2 Critical flows

**Flow A — Full Page workspace first impression (US-R1).**

1. Operator opens `/full-page` from Cloud Portal.
2. Plumes start drifting at t=0 (28s loop, reduced-motion: paused static gradient).
3. Workspace hero kinetic letter-reveal animates over 800ms (28ms stagger per character; reduced-motion: instant final state).
4. Stat strip count-ups fire on intersect (1400ms, `easeOutCubic`; reduced-motion: lands at final value).
5. Operator scans hero copy, sees `"Eight active maps, all healthy."` — reads as premium, marketing-grade.
6. Operator clicks `View activity` decorative CTA → Sonner toast: `"Activity log coming in a follow-on release. For now, the Dashboard Widget shows the most recent change timestamp."` per ADR-0030.

**Flow B — Context Panel inline quick-add (US-R5 — single-match case).**

1. Operator is in Pages editing `/products/sneaker-cloud-runner`. Context Panel mounts on the right side.
2. `subscribePageContext` fires → `pageInfo.url = "/products/sneaker-cloud-runner"`.
3. `matchPageRedirects` returns `matchedGroups.length === 1`. Form enters single-match state.
4. Hero count header renders `"1 redirect points here."` (note: grammar — `"redirects"` for N != 1, `"redirect"` for N == 1; design-spec commits to plural-aware copy).
5. Form source input is pre-populated with `/products/sneaker-cloud-runner` in mono font, editable.
6. RedirectType select renders disabled, pre-set to the matched map's redirectType (e.g. `301`), with hint `"uses Black Friday 2025's redirect type"` beneath.
7. Operator edits source to `/sneaker-cloud-runner-v2` and clicks `Add`.
8. `updateRedirectMap` fires with the matched map's existing attrs + appended mapping. Toast: `"Redirect added to Black Friday 2025."`. Form clears (source resets to `pageInfo.url`); matcher refetches; new row appears beneath the form with a brief highlight pulse (reduced-motion: instant).

**Flow C — Context Panel inline quick-add (US-R5 — multi-match case).**

1. Same as Flow B steps 1-2, but `matchedGroups.length === 3` (3 maps each have a mapping for this page).
2. Multi-match dropdown appears above the source input, label `"Adding to:"`, value `"Black Friday 2025 (most recent — 14 m ago)"`. Two more maps listed in the dropdown options sorted by `updatedAt` desc.
3. Hero count header: `"3 redirects point here."`. Three matched rows render beneath the form (each with parent-map name visible).
4. Form's RedirectType select is disabled, bound to the currently-selected map's redirectType.
5. Operator opens the dropdown → selects `"Q1 Product Launches (3 d ago)"`. Disabled RedirectType select re-binds (the new map's type renders). Hint copy updates: `"uses Q1 Product Launches's redirect type"`.
6. Operator types source, clicks `Add`. `updateRedirectMap` fires against the *selected* map. Toast confirms.

**Flow D — Context Panel inline quick-add (US-R5 — no-match / create-new case).**

1. `matchedGroups.length === 0`. No matched rows render. Form enters create-new state.
2. Hero count header: `"0 redirects point here."` with subline guidance copy beneath the count: `"Add the first redirect to start a map for this page."`.
3. Form's RedirectType select is **enabled** (default `Redirect301`). Multi-match dropdown does NOT render.
4. Below the source input, an auto-name preview appears in monospace `--muted-foreground` micro-copy: `"New map: sneaker-cloud-runner-redirects"` (per ADR-0029 auto-name pattern).
5. Operator picks RedirectType, types optional override source, clicks `Add`.
6. `createRedirectMap` fires with auto-name + default flags + form RedirectType + first mapping. Toast: `"Created sneaker-cloud-runner-redirects with one redirect."`. Surface re-renders into single-match state.

**Flow E — Dashboard Widget at-a-glance read (US-R3).**

1. Operator opens Cloud Portal home. Dashboard tile mounts.
2. Preview Data banner renders at top: `"Some metrics on this surface use preview data — wired up in a follow-on release."`.
3. Hero stat count-up `12,428` animates over 1400ms (reduced-motion: lands instantly).
4. Sparkline bars + top-destinations bar-fills animate on intersect (reduced-motion: render at final % instantly).
5. Operator scans the surface: hero number first, then sparkline trend, then "Last publish 14m ago by Anna" + green "all healthy" badge.
6. Operator clicks `Open` → routes to `/full-page`.

### 3.3 Responsiveness

| Surface | Primary design width | Smaller-width behavior | Larger-width behavior |
|---|---|---|---|
| Full Page | 1280px | At 768-1023px, hero stays full-width but stat strip collapses to 2 cols (2x2). Below 768px, left rail collapses above main column; hero clamp shrinks via `vw` term. | At ≥1440px, max-content-width caps at 1440px centered; hero clamp tops out at 96px. |
| Context Panel | 360px | Honors 320-400px iframe range. Below 320px (rare), inputs wrap their adornments; multi-match dropdown stays full-width. | Above 400px, panel content remains constrained at 360px effective content width (panel left-aligned). |
| Dashboard Widget | 480px | Below 400px (compact dashboard layouts), sparkline shrinks proportionally; top-destinations bar fills shrink to `6rem` max. | Above 600px (roomy dashboards), tile grows but hero stat caps at 80px; whitespace expands. |

Breakpoints inherited from PRD-000 Tailwind v4 Nova preset (`@theme inline { --breakpoint-md: 768px; --breakpoint-lg: 1024px; --breakpoint-xl: 1280px; }`). No new breakpoints introduced.

## 4. Component and pattern spec

### 4.1 Component inventory (refined POC + production)

For each component below, **states** are listed explicitly. Production component locations + the corresponding `@blok/*` primitive(s) are cited so the Lead Developer can wire them without guessing.

#### 4.1.1 `PreviewDataBanner` (NEW per ADR-0025)

- **Production path:** `site/components/ui/preview-data-banner.tsx`.

### 4.3 Accessibility notes (component-level)

- **All decorative motion**: `prefers-reduced-motion: reduce` honored at both the CSS layer (`@media`) and the JS layer (early-return in `script.js` count-up + letter-reveal helpers). Static fallbacks render the final state — never an empty container.
- **Focus-visible:** carries PRD-000 contract — 2px solid `var(--ring)` outline + 2px offset + `var(--radius-sm)` rounded. Applied automatically via global `:focus-visible` rule (theme.css).
- **Color contrast:** every color expression composes Blok tokens. The `--primary-foreground` dark-mode override (theme.css line 162, 202) is preserved verbatim — ensures the QuickCopy v0.1 readability fix carries forward.
- **Gradient-clip text fallback:** `@supports not (background-clip: text)` rule provides solid `var(--primary)` fallback. Browser floor (Chromium ≥85, Safari ≥14, Firefox ≥103) ships `background-clip: text` to >97% of evergreen users.
- **ARIA on multi-match dropdown:** `@blok/select` ships with proper `aria-label`, `role="combobox"`, `aria-expanded`. Custom label `"Adding to:"` rendered as `<label htmlFor="multi-match">`.
- **Screen-reader-only labels:** the `→` arrow in URL pairs is rendered as a `<span aria-hidden="true">→</span>` with adjacent `<span class="sr-only">redirects to</span>` for assistive tech.
- **Plume backdrop:** `aria-hidden="true"` on the `.fp-plume-backdrop` container — purely decorative.
- **Skeleton placeholders:** during initial load, `@blok/skeleton` renders at the exact dimensions of the final content so layout doesn't shift on resolve.

## 5. Responsiveness and accessibility

### 5.1 Breakpoints (Tailwind v4 Nova preset — carry from PRD-000)

| Token | Value | Use |
|---|---|---|
| `--breakpoint-md` | 768px | Stat-strip 2-col → 4-col; nav-links show in topbar |
| `--breakpoint-lg` | 1024px | Full Page rail-and-main two-column layout activates |
| `--breakpoint-xl` | 1280px | Primary design width for Full Page |

Context Panel and Dashboard Widget are extension-point iframes — their effective width is the iframe width, NOT the viewport. Per `sitecore:blok-marketplace-integration` "Iframe viewport constraints", treat panel and widget shells as mobile-first and design at the canonical widths (360px / 480px). Inside the iframe, `--breakpoint-md` / `--breakpoint-lg` likely never fire — but the layouts still behave correctly via flex/grid intrinsic sizing if the host stretches the iframe wider.

### 5.2 WCAG 2.1 AA — explicit checklist

- **1.4.3 Contrast (Minimum) AA** — every text/background pair against tokens checked:
  - `--foreground` on `--background` (light + dark): passes AA at all sizes.
  - `--muted-foreground` on `--background` (light + dark): passes AA for body text and AA Large for small text (verified PRD-000).
  - `--primary` (`#6e3fff` light / `#d9d4ff` dark) on `--background`: passes AA in light; passes AA in dark with `--primary-foreground` flip (`--color-blackAlpha-900`).
  - Gradient-clip text: both gradient stops are `--primary` derivatives, each passes AA against `--background` independently → mid-gradient also passes.
- **1.4.11 Non-text Contrast AA** — focus rings + interactive component edges:
  - Focus ring `var(--ring)` (`#6987f9`) against any background: passes AA Non-text contrast at 3:1.
  - `@blok/badge` borders are `color-mix` of token + transparency at 22-35% — verified visually distinct from card background.
- **2.1.1 Keyboard** — all interactive surfaces reachable via Tab. Multi-match dropdown opens with Space/Enter, navigates with Up/Down, closes with Escape (`@blok/select` defaults).
- **2.4.7 Focus Visible** — every focusable element has the global `:focus-visible` style.
- **2.5.5 Target Size AAA (best effort)** — primary action buttons ≥40px hit area on Context Panel touch surfaces (where iframe might be on iPad). `@blok/button` defaults are 32px height which is below AAA but meets AA-norm; one explicit oversize: the QuickRedirectForm Add button is `--blok-btn--sm` at 28px — keep as inline-form-appropriate; document as known sub-AAA trade-off.
- **2.3.3 Animation from Interactions** — `prefers-reduced-motion: reduce` honored everywhere.
- **3.3.2 Labels or Instructions** — QuickRedirectForm has visible `<label>` per field + helper hint for disabled-RedirectType-select case.
- **4.1.2 Name, Role, Value** — all controls expose name + role; disabled-RedirectType-select uses `aria-disabled="true"` (not the native `disabled` attribute — important for screen readers to announce the read-only-display semantics rather than skip the field).

### 5.3 Reduced-motion gating

Every animation must have a paired gate. The 5 animation surfaces:

| Animation | Driver | Reduced-motion fallback |
|---|---|---|
| Drifting plume backdrop (28s loop) | CSS `@keyframes backdrop-drift` on `.fp-plume-backdrop::before` + `::after` | `animation: none;` — static gradient persists |
