> **Reading note.** This is variant 3 of 3. v3 commits to a **navigable-map metaphor**: redirects are a *graph* of the site's URL surface — Redirect Maps are container nodes, mappings are directed edges (source → target). The metaphor lives in **composition, motion, and information architecture**, not in token values: every color, font, radius, and shadow is a Blok semantic token. The Site Atlas direction is opinionated; it leans on bolder typography hierarchy, generous accent usage, illustrative-but-token-only empty states, and a small spatial accent (mini-graphic on the Dashboard widget; optional minimap on the Full Page header). All spatial elements degrade to lists for keyboard and screen-reader users (NFR-A1..A5).

---

## 1. Design context & assumptions

### Audience

**Hybrid by surface** — applied through the Site Atlas metaphor:

- **Context Panel** — primary audience: marketers and content authors in Pages (PRD § 4 primary). Tone leans soft. The "atlas" surfaces here as **directional language and arrows** (`/foo` → `/bar`), not as graph nodes — the panel is a narrow strip and a node-graph would be claustrophobic. The metaphor manifests in the *visual hierarchy*: this page is a node; the displayed Redirect Maps are the edges that touch it; the parent Map is the container of the edge set.
- **Dashboard Widget** — primary audience: site managers (PRD § 4 primary). Tone is neutral. Tiles-first; the metaphor surfaces as a **tiny node-and-edge mini-graphic** in the corner of the card, giving an at-a-glance "shape" of the site's redirect set.
- **Full Page** — primary audience: Sitecore implementers and admins (PRD § 4 primary). Tone leans dense, technical. The metaphor surfaces strongest here: **breadcrumb minimap header** (collection → site → N maps), **directional arrow rows** for source→target, and an **expandable atlas-tree pattern** for the redirect list (parent Map row reveals child mapping rows below).

The Site Atlas direction is the **same north star** across all three surfaces; the *intensity* of the metaphor is calibrated to the audience.

### Goals and success criteria

Tied to PRD §§ 3 + 6 + 8:

- **G1 (Pages → Content Editor jump eliminated)** — the Context Panel reads as "this URL is touched by these redirects" with directional iconography that is immediately legible to a non-technical marketer. Group-by-Map (PRD US-1 AC) is explicit and visual.
- **G2 (per-site overview)** — the Dashboard Widget's three tiles answer "how many maps, how many edges, last touched when?" in ≤2s (NFR-P3). The mini-graphic is a glanceable accent — never a load-bearing data display.
- **G3 (single management surface)** — the Full Page is a power-user atlas: pick collection → pick site → expand maps → operate on edges. Two-pane (US-5) with tabbed fallback at narrow widths.
- **G4 (portable rule sets)** — Import/Export buttons sit at the surface root (Full Page header). Conflict resolution UX uses a per-row action picker with collapsible diff (PRD US-10 + ADR-0006).
- **NFR-A1..A5 (WCAG 2.1 AA)** — every spatial element has a list-mode equivalent reachable via keyboard alone; the minimap and mini-graphic are decorative SVG with `aria-hidden="true"` and a parallel text counter ("3 maps, 47 mappings"); the redirect-list expand pattern is keyboard-operable via the Blok `@blok/collapsible` semantics.

### Constraints (brand, a11y, stack)

- **Stack** — Next.js App Router (per ADR-0002), Mode A `MarketplaceProvider` singleton (per architecture § 2.3), Blok `@blok/*` registry primitives only (per architecture § 7); `react-virtuoso` for list virtualization (per ADR-0012); `react-hook-form + zod` for forms; `@dnd-kit/core` for mapping reorder.
- **Design system** — Blok semantic tokens only. Allowed tokens (verified via `sitecore:blok-theming`): `bg-background`, `bg-card`, `bg-muted`, `bg-primary`, `bg-destructive`, `bg-accent` (if Nova preset exposes; fallback: `bg-primary` + iconography), `text-foreground`, `text-muted-foreground`, `text-primary`, `text-primary-foreground` (verify pair on dark — known Nova trap, see Pitfalls), `border-border`, `border-destructive`, `ring-ring`, `shadow-sm`. **No hand-picked hex anywhere.** No font families beyond what `theme.json` ships (Geist Sans + Geist Mono via Nova preset — see `blok-setup` § Fonts). Custom illustrations and the minimap construction technique below are pure SVG using `currentColor` and `var(--blok-*)` token references; they do not introduce hex.
- **Iframe constraints** (per `sitecore:blok-marketplace-integration`):
  - Context Panel: 320–400px × variable (scrollable). No topbar; compact primitives.
  - Dashboard Widget: 300–800px × 200–400px. `@blok/dashboard-widget` outer frame.
  - Full Page: 1024–1920px × 600–1080px. `@blok/topbar` + two-pane content. Tabbed fallback below 1024px.
  - Shadows ≤ `shadow-sm` on app-shell surfaces (large shadows clip at iframe edges).
- **A11y target** — WCAG 2.1 AA. Keyboard reachable, focus-visible via Blok `ring-ring`, `aria-live="polite"` on save/error toasts, `aria-expanded` on collapsible rows, `aria-pressed` on toggles, `role="status"` on banners. Spatial accents are decorative (`aria-hidden="true"`) with parallel text data.
- **Light + dark parity** — Blok `theme.json` flips automatically under `.dark`; no manual dark-mode work beyond honoring tokens. **Verify `--primary-foreground` against `--primary` on dark before relying on `bg-primary text-primary-foreground`** (Nova has shipped them collapsed once — see Pitfalls).

### Reference / inspiration

Adapted, never copied — synthesizing the spatial metaphor onto a Blok-grounded surface:

- **Linear's project-graph view** — directional arrow chips for issue links; the arrow itself is a meaningful signifier, not decoration. Adapted: source→target arrows in the Context Panel and Full Page rows.
- **Vercel's deployment-tree breadcrumb** — small route-shape indicators in the topbar that show structural context without taking up real estate. Adapted: the optional minimap header on Full Page (collection → site → N maps).
- **Notion's database "tree" view** — parent rows expand to reveal child rows in a single visual unit; the relationship is preserved when scrolling because children indent under their parent. Adapted: the Full Page redirect list pattern (Map → mappings, all in one expandable scope).
- **Stripe's "test mode" banner pattern** — a persistent, non-dismissible status banner that never blends into chrome. Adapted: the Context Panel "Direct-string matches only" banner (PRD US-1 AC, ADR-0005).
- **MapBox's static-map placeholder graphics** — minimal SVG node-and-edge illustrations that read as "shape of a network" without being live data. Adapted: the Dashboard Widget mini-graphic (decorative; paired with text counters that are the actual answer).

### Answers to clarifying questions (variant-specific)

- **Q: How literally do we render the "atlas"?** — A: As an *information-architecture* metaphor first, and a *visual* metaphor second. The IA (parent Map → child mappings, source → target arrows) is load-bearing; the visuals (minimap, mini-graphic) are decorative accents that pair with text.
- **Q: Do we use a real interactive node graph anywhere?** — A: No. Interactive force-directed graphs would inflate bundle size, fail iframe-narrow widths, and cost a11y points. The "atlas" is read through *typographic + iconographic + IA cues*, not WebGL.
- **Q: How prominent are arrows?** — A: Very. The source→target arrow is the visual signature of this variant. It's an inline SVG icon (Lucide `ArrowRight` or equivalent) at every mapping row in every surface.
- **Q: How dense is the Full Page?** — A: Standard density (Blok defaults). Site Atlas adds *visual hierarchy* (bold display headings, accent color on the active site, accent borders on expanded rows) — not extra rows-per-screen.
- **Q: Where is the minimap on Full Page?** — A: A header strip above the redirect list; collapses to a single breadcrumb line below 1280px iframe width and disappears entirely below 1024px.
- **Q: Empty states — illustrative or austere?** — A: Subtly illustrative — a small Blok-token SVG of a map with no nodes, paired with calm copy. Never busy, never branded with hex colors.

### Assumptions made (label-friendly)

- **A-V3-1** — Marketers reading the Context Panel benefit from explicit `source → target` arrow rows more than from a label-pair format. The arrow is an unambiguous directional cue. Validation: 5-min editor walkthrough (PRD m5).
- **A-V3-2** — The Dashboard mini-graphic adds joy without confusing site managers because the three tiles carry the answer; the graphic is paired with `aria-hidden="true"` and a parallel `<span class="sr-only">`. Validation: smoke at /test in light + dark.
- **A-V3-3** — The Full Page minimap header is *useful at first paint* (orientation: "I'm in collection X, site Y, with N maps") but does not replace the picker. The picker remains the primary navigation. Validation: usability check during smoke.
- **A-V3-4** — At 30 maps × 500 mappings (NFR-S1), the expandable parent-child pattern remains traceable because (a) only one map's children are expanded at a time by default, (b) `react-virtuoso` virtualizes the parent list, (c) child mapping rows render inline under their expanded parent, never in a parallel column. The long-list relationship-visibility check therefore passes — see § 5 explicit answer.
- **A-V3-5** — The "Site Atlas" identity is robust enough that Blok semantic tokens carry it without custom palette additions. The accent moments come from `bg-primary` selectively applied (active site row, expanded map left-border, primary CTA buttons), not from new colors.
- **A-V3-6** — Marketers do not need the minimap on the narrow Context Panel. The metaphor surfaces there only as language ("redirects affecting `/foo/bar`") and arrows. Validation: heuristic review.

## 2. Design direction

### Visual direction — "Site Atlas"

A **navigable map of the site's URL surface**. Every mapping is a directed edge; every Redirect Map is a container of edges. The visual signature is the **source → target arrow** — bold, monospaced URL paths flanking a `text-primary` arrow glyph, repeated across all three surfaces.

Typography rhythm is **bolder than v1/v2**: display-weight headlines on the Full Page topbar and on Context Panel section headers; monospace (Geist Mono) for URL paths because they *are* technical strings and reading them as monospace makes typos visible. Body copy stays Geist Sans, body weight.

Color is **deliberately accented**: most surface area is `bg-background` + `text-foreground`; `bg-primary` is reserved for (a) the active site row's left border, (b) the expanded Redirect Map's left border, (c) primary CTA buttons (Save, Confirm Import), (d) the breadcrumb minimap's "you-are-here" node, (e) the loading-state progress underline. `bg-muted` is used for the inactive scope of the minimap and for inline `RedirectType` badges. `bg-destructive` is reserved for delete-confirm and the "missing datasource" warning border. `border-l-4 border-primary` is the Site Atlas signature treatment for "this is the active container" — it appears in three places consistently.

Motion is **directional**: expand-collapse uses a 200ms ease-out vertical reveal (CSS `max-height` transition with `prefers-reduced-motion: reduce` fallback to instant). The arrow glyph in source→target rows has a 120ms `translate-x-0.5` micro-shift on hover (purely decorative; reduced-motion disables). The minimap "you-are-here" node has a 1.4s `prefers-reduced-motion`-respecting subtle pulse (`opacity-70 → opacity-100`) — and is `aria-hidden="true"`.

The variant's name comes from how the Full Page reads at first glance: a **header that maps the surface** (collection → site → N maps), then a **tree of containers and edges**. The Atlas is metaphorical, not literal: there is no zoomable map, no canvas, no dragging-of-nodes. The IA is the atlas.

### Layout and structure

#### Context Panel — `xmc:pages:contextpanel` (route `/context-panel`, viewport ~360px wide)

Vertical-stack, panel-shell pattern (no topbar — too tall for the panel; per `blok-marketplace-integration` table). Top-down rhythm:

1. **Persistent regex banner** (per PRD US-1 AC, ADR-0005) — a `@blok/alert` with `variant="warning"` (or `variant="default"` styled with `border-l-4 border-primary` if the Nova preset doesn't expose `warning`). Copy: *"Direct-string matches only — regex pattern matches are not yet covered."* `role="status"`. **Non-dismissible.** Lives at the top of the panel, scrolls with content.
2. **Page-context heading** — Blok `text-base font-semibold text-foreground`. Reads:
   *"Redirects affecting"* — small label.
   `/foo/bar` — Geist Mono, `text-base`, `text-foreground`, full-width on its own line so long paths wrap legibly.
3. **Counter sentence** — `text-sm text-muted-foreground`:
   *"**3** Redirect Maps · **5** mappings"* — counter pills are `@blok/badge` `variant="default"`, `colorScheme="muted"`. If 0, this sentence is replaced by the empty-state (see § 4 states).
4. **Add-redirect CTA** — `@blok/button` `variant="primary"` `size="default"`, full width. Label: *"+ Add redirect for this page"*. Opens the Add Redirect modal (described below).
5. **Group-by-Map list** — vertical stack of group cards (`@blok/card` with `p-3` `gap-2`). Each group card represents one parent Redirect Map; structure:
   - **Map header row** (full-width, top of card):
     - Map display name — `font-semibold text-sm text-foreground`.
     - `RedirectType` badge — `@blok/badge` `variant="bold"` `colorScheme="primary"` (`301`, `302`, `307`, `Server Transfer`).
     - Flag chips — only rendered when the flag is true. Three possible chips: `Q-string preserved`, `Lang preserved`, `Vfolder included`. `@blok/badge` `variant="ghost"` `colorScheme="muted"` `size="sm"`.
   - **Mapping rows beneath the header** — each matched mapping is one row, `border-t border-border` separator (skipped on first row). Row layout:
     - Source URL — Geist Mono, `text-sm`. If this matches the current page (the page is the *source* of the redirect), the URL is rendered in `text-foreground font-medium`.
     - Arrow glyph — Lucide `ArrowRight` icon, 16px, `text-primary`.
     - Target URL — Geist Mono, `text-sm`. If this matches the current page (the page is the *target*), the URL is rendered in `text-foreground font-medium`.
     - Match-direction chip — small `@blok/badge` `size="xs"` reading `from this page` (when source matches) or `to this page` (when target matches). Helps the marketer understand what role the page plays in this rule.
     - Edit / delete kebab menu — `@blok/dropdown` triggered by an icon button (Lucide `MoreHorizontal`). Menu items: *Edit*, *Delete*. Edit opens an inline form; Delete opens a confirm `@blok/alert-dialog`.
6. **Add Redirect modal** — `@blok/dialog`, sized to fit the panel viewport (≤90% of iframe width). Two-step:
   - **Step 1: Pick a Map** — `@blok/search-input` plus a list of existing Maps in the current site (one row each: name + `RedirectType` badge + last-updated relative timestamp). The first row is fixed: *"+ Create new Redirect Map"* with a Lucide `Plus` icon.
   - **Step 2a (existing Map picked)**: inline form with two Geist Mono input rows — Source (pre-populated with the current page URL, editable) and Target (required). Save button: `@blok/button` `variant="primary"`. Cancel: `variant="ghost"`.
   - **Step 2b (Create new Map picked)**: form with — Map name (`@blok/input`), `RedirectType` (`@blok/select`, **initial value empty**, options derived from real-tenant introspection per architecture § 4.4), three flag checkboxes (`@blok/checkbox`), then the same Source/Target row as 2a. Save validates non-empty source, non-empty target, non-empty `RedirectType` (PRD FR-11) and shows inline `text-destructive` error sentences.

**Long-list relationship-visibility check (per agent identity § 92):** the panel is **already nested by design** — mappings are children of their parent Map card, never in a parallel column. At 20+ Maps × 20+ mappings the user scrolls one continuous list; the Map header always sits directly above its mappings. The check **passes**.

#### Dashboard Widget — `xmc:dashboardblocks` (route `/dashboard-widget`, viewport 300–800 × 200–400)

Bounded card with `@blok/dashboard-widget` outer frame. Composition:

1. **Header row**:
   - Widget title — Blok `text-sm font-semibold text-muted-foreground`: *"Redirects · `<site name>`"*.
   - Optional kebab — link out to Full Page (`@blok/dropdown`, single item: *"Open in Redirect Manager"*).
2. **Stat-tile row** — three tiles in a CSS grid (`grid-cols-3 gap-3`). Each tile is a `@blok/card` with `p-3`:
   - **Tile 1 — Maps**: large numeral (Blok `text-2xl font-bold text-foreground`), label below in `text-xs text-muted-foreground`: *"Redirect Maps"*.
   - **Tile 2 — Mappings**: same pattern, label: *"Mappings"*.
   - **Tile 3 — Last updated**: top line in `text-2xl font-bold text-foreground` — relative timestamp (`2h ago`, `3d ago`, `Apr 27`); label below: *"Last updated"*.
3. **Spatial accent — mini-graphic** (optional, decorative) — a small SVG positioned in the **top-right corner** of the widget header, ~64×40px. Construction technique:
   - Pure inline SVG, `aria-hidden="true"`, no raster, no external font.
   - Three small circles (`<circle r="3" fill="currentColor">`) representing Map nodes, connected by two thin lines (`<line stroke="currentColor" stroke-width="1.5" stroke-opacity="0.5">`) representing mapping edges. The number of nodes scales from 1 to 3 based on `Math.min(3, mapsCount)` — purely indicative shape, never load-bearing data.
   - The wrapping element has `text-muted-foreground` (so `currentColor` resolves to the muted Blok semantic token, dark-mode-safe).
   - **Accessible alternative:** the tile-row ALREADY carries the actual numbers; the SVG is decorative. A `<span className="sr-only">` adjacent reads *"Three Redirect Maps in this site."* for screen-reader users who happen to land on it.
4. **Footnote line** (PRD US-4 AC) — full-width below tiles, `text-xs text-muted-foreground`: *"Redirect counts only — usage analytics ship in a follow-on release."*

**Sites with zero Maps** — empty state replaces the tile row:

- Decorative SVG, ~80×60px, `text-muted-foreground` `currentColor`: a faint outline of a map frame with one large dashed circle node at center (token-only).
- Heading: *"No redirects configured for this site."* (`text-sm font-medium`).
- One CTA button: *"Open Redirect Manager"* — links to Full Page route.
- The footnote line is suppressed in this state.

#### Full Page — `xmc:fullscreen` (route `/full-page`, viewport 1024–1920 × 600–1080)

Two-pane with optional minimap header. Composition:

1. **Topbar** — `@blok/topbar` shell with:
   - App name on the left: *"Redirect Manager"* (`text-base font-semibold`).
   - Action cluster on the right:
     - `@blok/button` `variant="ghost"` *Import* — opens the import flow (described below). Lucide `Upload` icon.
     - `@blok/button` `variant="ghost"` *Export* — triggers download of `redirects-<site>-<timestamp>.json` (per PRD US-9). Lucide `Download` icon. **Disabled when no site is selected.**
     - `@blok/button` `variant="primary"` *+ New Redirect Map* — opens the Map create form. Lucide `Plus` icon. **Disabled when no site is selected.**
2. **Optional minimap header strip** — full-width below topbar; visible at iframe widths ≥1280px; collapses to a single breadcrumb line at 1024–1280px; hidden below 1024px. Construction:
   - **Width ≥1280px** — three nodes connected by `→` glyphs:
     - Node 1: collection name, `@blok/badge` `variant="ghost"` `colorScheme="muted"` if not active, or `colorScheme="primary"` with `border-l-4 border-primary` if it's the surface root.
     - Node 2: site name, same treatment, becomes "you-are-here" with the `bg-primary text-primary-foreground` chip when a site is selected.
     - Node 3: count chip — `text-sm text-muted-foreground` reading *"**N** Redirect Maps · **M** mappings"*. The numerals are `text-foreground font-medium`.
     - The `→` glyphs are Lucide `ChevronRight` icons, `text-muted-foreground`.
     - The "you-are-here" node has the gentle 1.4s opacity-pulse described in § Motion (reduced-motion safe; `aria-hidden="true"`).
   - **Width 1024–1280px** — the minimap collapses to a single breadcrumb line: `Collection / Site · 12 maps`. Same tokens, no node styling, no animation.
   - **Width <1024px** — minimap suppressed entirely; the picker becomes the primary navigation.
   - **A11y alternative:** the entire minimap is `aria-hidden="true"`; the parallel `<h2 className="sr-only">` reads *"In collection X, site Y, viewing N Redirect Maps."* The `→` glyphs are decorative; the data is in the heading.
3. **Two-pane main area**:
   - **Left pane (~320px wide, fixed)** — Picker. Stacked:
     - `@blok/select` Collection — `aria-label="Site collection"`. Empty state: *"Pick a collection"*.
     - `@blok/select` Site — `aria-label="Site"`. Disabled until a collection is picked. Sites filtered to the chosen collection.
     - **Active-site indicator** — when a site is picked, a `@blok/card` below the selects shows: site display name (`font-semibold`), site hostname (Geist Mono `text-xs text-muted-foreground`), language badge (`@blok/badge` `colorScheme="muted"` reading `en` — MVP is `en`-only per ADR-0010). The card has the Site Atlas signature treatment: `border-l-4 border-primary`.
   - **Right pane (flex-1)** — content. State machine:
     - **No collection picked** — Blok empty state: decorative map-frame SVG + heading *"Pick a collection to begin"* + body sentence *"Pick a site collection on the left, then a site, to see its redirects."*
     - **No site picked** — same pattern, copy: *"Pick a site to see its redirects."*
     - **Loading** — Blok `@blok/skeleton` rows (3 placeholder Map rows).
     - **Site has 0 Maps** — empty state: decorative SVG (faint map outline, dashed center node) + heading *"No Redirect Maps in this site"* + body *"Create the first one to start managing redirects."* + CTA `@blok/button` `variant="primary"` *+ New Redirect Map*.
     - **Loaded with Maps** — the redirect list (next).
4. **Redirect list** (right pane content) — virtualized expandable tree using `react-virtuoso` (per ADR-0012). Each parent row is one Redirect Map; expanded rows reveal child mapping rows inline beneath the parent.
   - **Parent row layout** (collapsed default):
     - Lucide `ChevronRight` (collapsed) / `ChevronDown` (expanded) — small icon button left.
     - Map name — `font-semibold text-sm text-foreground`.
     - `RedirectType` badge — `@blok/badge` `colorScheme="primary"` (`301`, `302`, `307`, `Server Transfer`).
     - Mapping count — `@blok/badge` `colorScheme="muted"` `text-xs`: *"`12` mappings"*.
     - Flag chips — same three as Context Panel (only rendered when true).
     - Last-updated — `text-xs text-muted-foreground` right-aligned: *"Updated 2h ago"*.
     - Edit / delete kebab — `@blok/dropdown` triggered by Lucide `MoreHorizontal`. Items: *Edit Map*, *Delete Map*.
   - **Expanded row** — gains `border-l-4 border-primary` along its left edge (Site Atlas signature). Reveals:
     - Mapping rows (one per `{source, target}` row from parsed `UrlMapping`):
       - Drag handle — Lucide `GripVertical`, `text-muted-foreground` (per `@dnd-kit/core` reorder, US-8).
       - Source — Geist Mono `text-sm text-foreground`. Truncates with title-tooltip on overflow.
       - Arrow — Lucide `ArrowRight`, 16px, `text-primary`.
       - Target — Geist Mono `text-sm text-foreground`. Same truncation.
       - Edit / delete icons — Lucide `Pencil` / `Trash2` icon buttons, `text-muted-foreground hover:text-foreground`.
     - Below the last mapping row: `+ Add mapping` ghost button (full width of the inner area, `border-dashed border-border`, `text-primary`).
5. **Map create / edit form** — opens as a `@blok/sheet` (right-hand drawer, ~480px wide; falls back to a `@blok/dialog` below 1024px). Fields:
   - Map name — `@blok/input` (required).
   - `RedirectType` — `@blok/select`, **initial value empty** (per PRD FR-11), options derived from real-tenant introspection.
   - Three flag checkboxes — `@blok/checkbox` × 3.
   - Mappings repeater — same row layout as the expanded redirect list (drag handle, source, arrow, target, delete) with `+ Add mapping`.
   - Footer: `@blok/button` `variant="primary"` *Save* + `variant="ghost"` *Cancel*. Save runs inline validation (FR-11). Cancel triggers the dirty-confirm pattern (PRD FR-12).
6. **Delete confirm** — `@blok/alert-dialog`. Heading: *"Delete `<map name>`?"*. Body: *"This will remove the Redirect Map and all its mappings. This cannot be undone."* Buttons: `Cancel` ghost; `Delete` destructive (`bg-destructive text-destructive-foreground`).
7. **Friendly error UX** (PRD FR-13) — on any write failure, render `@blok/alert` `variant="destructive"` at the top of the right pane:
   - Heading: *"Couldn't save this Redirect Map"*.
   - Sub-line: a friendly translation of the error class (e.g. *"Sitecore rejected the request. You can retry, or check the technical details."*).
   - `@blok/collapsible` *"Show technical details"* toggle revealing the verbatim GraphQL error in a `<pre>` block with Geist Mono `text-xs` (`bg-muted` background; `text-muted-foreground` text).
   - Two buttons: `Retry` (`variant="primary"`) + `Dismiss` (`variant="ghost"`).
8. **Cancel/discard with confirm-on-dirty** (PRD FR-12) — when the operator closes the create/edit Sheet with unsaved changes, render `@blok/alert-dialog`: *"Discard your changes?"* with `Discard` (`variant="destructive"`) and `Keep editing` (`variant="ghost"`). Pristine forms close silently.

**Long-list relationship-visibility check (mandatory per agent identity § 92):**
*Question:* at 20+ items in either layer (Redirect Maps + their mappings), can the user trace which mapping belongs to which Map?
*Answer:* **YES — and trivially so**, because the design uses a **single nested list**, not two parallel sections. Mappings render *inside* their parent's expanded scope, indented and with the parent's `border-l-4 border-primary` running down the left edge through the entire expanded block. There is no second column or parallel section that could become misaligned on scroll. The virtualization (react-virtuoso) preserves the parent-child grouping inside the virtualized window — a child row never appears separated from its parent. Worst case at 30 maps × 500 mappings (NFR-S1): the user scrolls a continuous virtualized list; one expanded map at a time is the default; expanding a second map collapses the first only when the operator chooses (no auto-collapse; collapse-on-expand-other is opt-in via a toolbar toggle, default off).
*Verified:* the Site Atlas metaphor *strengthens* the parent-child relationship rather than obscuring it — the `border-l-4 border-primary` is exactly the visual hint that says "all of these rows live under this Map".

#### Import flow (lazy-loaded `@blok/dialog` step sequence)

Triggered from the Full Page topbar Import button. Steps:

1. **Pick file** — `@blok/dialog`. Dropzone area (large dashed `border-border` rectangle) + `Browse...` button. On file pick → schema validation runs immediately.
2. **Schema error** (if validation fails) — close current dialog state and render: `@blok/alert` `variant="destructive"` inside the dialog, copy: *"Couldn't read this file. It must match `redirect-manager/v1`."* + the Zod parse error in a collapsible technical-details block.
3. **Preview screen** (heaviest UI — `next/dynamic`-loaded per architecture § 7) — full-screen replacement of the right pane:
   - **Header strip**: *"Importing **N** items into `<site>`"* + close X.
   - **Bulk control**: `@blok/select` *"Apply same action to all conflicts"* with options: *(none)*, *Create*, *Overwrite*, *Skip*.
   - **Item list** — virtualized table-like (Blok `@blok/table` wrapped in `@blok/scroll-area`), grouped:
     - **Section: New items** — header `text-sm font-semibold text-muted-foreground`. Each row: a Lucide `Plus` icon, item name, `RedirectType` badge, mapping count, action chip *Create* (`@blok/badge` `colorScheme="primary"`).
     - **Section: Conflicting items** — header `text-sm font-semibold text-foreground`. Each row:
       - Item name + `RedirectType` badge + mapping count.
       - **Per-row action picker** — three radio-styled buttons (composed from `@blok/button` set with `aria-pressed`, per `blok-components` § "toggle-group specifically"): *Create* / *Overwrite* / *Skip*. **No default selected.**
       - **Diff toggle** — `@blok/collapsible` trigger *"Show diff"*. When opened: `@blok/table` showing field-by-field changes (name, RedirectType, three flags, mapping list — added rows in `bg-primary/10 text-foreground`, removed rows in `bg-destructive/10 text-foreground`, unchanged in default; **also** prefixed with `+`/`-`/` ` glyphs so the signal is not color-only — NFR-A5).
   - **Footer**:
     - Status sentence: *"**X** of **Y** conflicts decided."* `text-sm text-muted-foreground`.
     - Buttons: `Cancel` (`variant="ghost"`) + `Confirm import` (`variant="primary"`, **disabled until every conflict has an action**).
4. **Apply progress** — replaces preview with a single-line narration pattern (similar to the Atlas's reading-state aesthetic):
   - *"Importing item **3** of **17**..."* `text-base text-foreground`, updates as each mutation resolves.
   - Thin `bg-primary` progress underline below.
   - `aria-live="polite"` on the narration line so screen readers track progress.
5. **Final summary** — full-width replacement:
   - Heading: *"Import complete"* (or *"Import finished with errors"* if any failed).
   - Two stat tiles: *"**N** succeeded"* (`bg-card`) and *"**M** failed"* (`bg-destructive/10` if M>0, `bg-muted` if M=0).
   - Per-item list (same table primitive) with status icons — Lucide `Check` (success, `text-primary`) or `X` (failure, `text-destructive`) plus the verbatim error message in a collapsible block.
   - "Retry failed items only" button (per PRD R4 mitigation) when M>0.
   - "Done" button returns to the redirect list.

### Key components and patterns

- **Source → target arrow row** — the variant's signature pattern. Geist Mono URL + `ArrowRight` glyph + Geist Mono URL. Used in Context Panel mapping rows, Full Page expanded mapping rows, and import diff rows. The arrow is `text-primary`; the URLs are `text-foreground`.
- **Atlas signature border** — `border-l-4 border-primary` along the left edge of the active container. Used on (a) the active site card in Full Page picker, (b) the expanded Redirect Map row in the redirect list, (c) the breadcrumb minimap "you-are-here" node. Three consistent appearances; nowhere else.
- **Group-card pattern** (Context Panel) — a header line with name + `RedirectType` badge + flag chips, then mapping rows beneath. Replaces a flat-list pattern.
- **Expandable tree pattern** (Full Page redirect list) — single nested list; parent rows expand inline; children indent with the parent's signature border running the full height of the expanded block. Replaces a parallel two-column "maps + mappings" pattern that would fail the long-list relationship check.
- **Decorative map graphics** — pure-SVG, token-only, `aria-hidden="true"`, paired with parallel text. Used on the Dashboard mini-graphic, the Dashboard empty state, and the Full Page empty states. Never renders live data.
- **Optional minimap header** — three-node `→`-connected breadcrumb on Full Page wide widths; degrades to a single text breadcrumb at 1024–1280px; suppressed below 1024px.
- **Persistent regex banner** (Context Panel) — non-dismissible `@blok/alert` at the top of the panel.
- **Friendly error banner with technical details** — `@blok/alert` `variant="destructive"` with a `@blok/collapsible` revealing the verbatim GraphQL error.
- **Confirm-on-dirty pattern** — `@blok/alert-dialog` triggered when closing a Sheet/Dialog with unsaved changes; pristine forms close silently.

### Accessibility notes

- **Keyboard ops** — every interactive element reachable via Tab; Enter/Space activates buttons; Escape closes Dialogs/Sheets/Dropdowns; arrow keys navigate Select options and within toggle-group radio actions in import preview.
- **Focus-visible** — Blok `ring-ring ring-2 ring-offset-2` on every focusable element. Never `outline: none` without replacement (NFR-A4).
- **Screen-reader labels** — all icon-only buttons have `aria-label` (e.g. `aria-label="Edit mapping"`, `aria-label="Delete mapping"`). The kebab triggers announce as `aria-label="Mapping options"`. The expand toggle uses `aria-expanded={isExpanded}`. The minimap is `aria-hidden="true"` with a parallel `<h2 className="sr-only">` carrying the data.
- **Live regions** — success toasts use `aria-live="polite"`. Import progress narration is `aria-live="polite"`. The regex banner is `role="status"`.
- **Color contrast** — all token combinations verified at QA per Blok theming guidance. Watch the `bg-primary text-primary-foreground` pair on dark mode (Nova has shipped them collapsed once — see Pitfalls).
- **Match-direction chip** — the `from this page` / `to this page` chip in the Context Panel is explicit text, not color-coded. Marketers don't need to interpret a color difference.
- **Diff color in import preview** — added rows use `bg-primary/10` *plus* a `+` glyph; removed rows use `bg-destructive/10` *plus* a `-` glyph. Color is reinforcement, not the only signal (NFR-A5).
- **Spatial accents (mini-graphic + minimap)** — `aria-hidden="true"`; parallel `<span className="sr-only">` or `<h2 className="sr-only">` carries the data. List-mode for the minimap is implicit: the picker on the left is the keyboard navigation; the minimap is a glanceable orientation aid only.
- **Reduced motion** — all transitions wrapped in `@media (prefers-reduced-motion: reduce)` blocks that disable transitions and animations (the chevron rotation, the arrow micro-shift, the minimap pulse, the import-progress underline animation).
- **Focus within Sheet/Dialog** — focus-trap inside; Escape closes; first focusable element auto-focuses on open (per Blok primitive behavior).

### Pitfalls to flag for the implementing developer (06)

- **`bg-primary text-primary-foreground` on dark mode** — Nova preset has shipped with `--primary-foreground` collapsed onto `--primary` once (`blok-theming` § Pitfalls). Verify the pair resolves to a contrasting value in **both** themes before relying on it for the active-site card, the expanded Map's signature border-adjacent text, and the `+ Add redirect` button. If dark-mode contrast fails QA, use `text-background` instead, or scope a local `--primary-foreground` override on the Site Atlas signature surfaces.
- **Color-emoji codepoints** — `❌` `✅` `⚠️` are CSS poison. This spec uses Lucide icons throughout (`X`, `Check`, `AlertTriangle`). Do not substitute emoji codepoints for state icons — they ignore `color: hsl(var(--destructive))` and look wrong on theme flip.
- **Tailwind v4 / Nova preset** — there is **no** `tailwind.config.ts`. Token additions go in `app/globals.css` `@theme inline { … }`. If a future iteration of this spec introduces a new spacing or radius token, edit globals.css, not a JS config (`blok-setup` § Tailwind v4 / Nova preset).
- **Quickstart-installed primitive subset** — the Marketplace quickstart only installs `alert, badge, button, card, collapsible, separator, skeleton`. Every other primitive listed in § 4 needs an explicit `npx shadcn@latest add @blok/<name>` post-scaffold. The implementing developer's first task is to install all primitives listed in the inventory; missing one will cause runtime import errors that won't surface at scaffold time.
- **`react-virtuoso` + nested expand** — virtualizing a parent-list while children render inline expanded under a parent is supported but requires `<TableVirtuoso>` or the `<Virtuoso>` `itemContent` to render the expanded block as part of the same item, not as a sibling. If naive virtualization is used, child rows can pop in/out as the parent scrolls — defeating the entire long-list relationship-visibility check. Verify on first render with 30 maps × 500 mappings.
- **`pages.reloadCanvas`** — fire after every successful Context Panel write so the Pages canvas reflects the redirect immediately (architecture § 5.6). Easy to forget — it's a one-line call but a UX-critical one.
